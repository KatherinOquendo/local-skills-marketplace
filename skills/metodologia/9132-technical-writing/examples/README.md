# Technical Writing — Example Outputs

Sample outputs demonstrating what the technical-writing skill produces when applied to a fictional "TransLogix S.A. Logistics Platform Modernization" project.

## Files

- `sample-output.md` — Markdown format (default output)
- `sample-output.html` — HTML branded format (MetodologIA Design System)

## Usage

These examples show an excerpt of a 03_Analisis_AS-IS document covering 5 sections of a monolithic Order Management System (OMS). The sample demonstrates all technical writing standards: progressive disclosure (TL;DR at document and section level), evidence attribution tags ([CODIGO], [CONFIG], [DOC], [INFERENCIA], [SUPUESTO], [STAKEHOLDER], [BENCHMARK]), terminology consistency, callout system, cross-references, anti-pattern compliance, and Mermaid diagrams with accessibility text.

## What to Look For

- **Progressive disclosure**: Each section opens with a TL;DR summary, followed by dense analysis, tables with evidence, and callouts.
- **Evidence tags**: Every quantitative claim and causal assertion carries a source tag.
- **Terminology consistency**: "monolito" is defined once and used consistently (never "sistema legacy" or "aplicacion principal").
- **Anti-patterns avoided**: Zero filler phrases, active voice with identified agent, no heading level skips, all tables have headers, all scores have justification.
- **Callouts**: Insight, Risk, Trade-off, and Evidence callouts used appropriately after findings.
- **Cross-references**: Each section links to related deliverables using the standard format.
- **FTE-months, not prices**: Effort estimates use FTE-months with explicit disclaimers.
