Section 2: Use Cases

UC-01 Abrir Cuenta Digital

Actor primarioCliente nuevo
PrioridadHigh
FrecuenciaDaily
PrecondicionesCliente no tiene cuenta activa. Dispositivo con camara para INE.

Flujo principal

  1. Cliente selecciona "Abrir Cuenta" en la app.
  2. Sistema solicita datos personales (nombre, CURP, RFC, correo, telefono).
  3. Cliente captura foto de INE (frente y reverso).
  4. Sistema ejecuta verificacion KYC con proveedor externo.
  5. Sistema valida CURP contra RENAPO.
  6. Sistema aplica BR-01 (edad minima) y BR-02 (duplicados).
  7. Sistema genera numero de cuenta y CLABE.
  8. Sistema envia correo de confirmacion con contrato digital.

Flujos alternativos

  • 4a. KYC rechazado por calidad de imagen: recaptura (max 3 intentos).
  • 6a. Menor de edad: mensaje de rechazo. Caso termina.

Excepcion

  • 4x. Proveedor KYC no disponible: encola solicitud, notifica procesamiento en 24h.

Postcondiciones: Cuenta ACTIVA creada. Cliente puede operar.

UC-02 Transferir Fondos via SPEI

Actor primarioCliente titular
PrioridadHigh
FrecuenciaDaily

Flujo principal

  1. Cliente selecciona "Transferir" y elige cuenta origen.
  2. Sistema muestra saldo disponible.
  3. Cliente ingresa CLABE destino, monto y concepto.
  4. Sistema aplica BR-03 (limites).
  5. Sistema solicita segundo factor.
  6. Sistema envia orden SPEI.
  7. Sistema confirma y actualiza saldo.
  8. Notificacion push y correo.

UC-03 Solicitar Prestamo Personal

Actor primarioCliente titular
PrioridadHigh
FrecuenciaSession

Flujo principal

  1. Cliente selecciona "Solicitar Prestamo".
  2. Sistema muestra productos segun perfil.
  3. Cliente selecciona producto, monto y plazo.
  4. Simulacion de pagos (monto, tasa, CAT, pago mensual).
  5. Cliente confirma solicitud.
  6. Motor de scoring (BR-04).
  7. Resolucion: aprobado / rechazado / documentacion adicional.
  8. Si aprobado: contrato digital + desembolso programado.
