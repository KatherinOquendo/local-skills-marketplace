## Usage

These examples demonstrate the full output of the `sofka-ux-writing` skill with `{VARIANTE}=tecnica` and `{FORMATO}=dual`.

**Scenario:** Full 5-standard audit of Acme Corp's architecture deep-dive document. The document has 12 sections, 8,500 words, and targets both executive and technical audiences.

## Key Sections Demonstrated

1. **Standard 1: Information Hierarchy** — Section structure assessment, 30/70 scanning ratio analysis
2. **Standard 2: Cognitive Load** — Chunking analysis, progressive disclosure gaps, terminology consistency
3. **Standard 3: Scannability** — 80/20 rule evaluation, heading hierarchy, callout usage
4. **Standard 4: Microcopy** — CTA review, error message patterns, empty state improvements
5. **Standard 5: Readability Heuristics** — Flesch-Kincaid scores, sentence length analysis, jargon density
