# Prompt: Plugin Architecture Planning

```
Design the architecture for plugin {PLUGIN_NAME}.

Based on the concept brief, define:
1. Agent roster: name, role, movements owned
2. Skill catalog: name, movement, purpose, MOAT depth
3. Command map: canonical commands + aliases, which skill each invokes
4. Hook strategy: which events need hooks, what type
5. Movement pipeline: sequence, quality gates, data contracts
6. Cross-reference matrix: which agent owns which skills

Output:
- Agent roster table
- Skill catalog table (organized by movement)
- Command matrix
- Pipeline diagram (text/Mermaid)
- Data contract between movements
- Estimated total file count
```
