Patrones de Context Mapping

  • Core Banking ↔ Digital Channels: Customer-Supplier (Core Banking upstream)
  • Core Banking ↔ Risk & Compliance: Shared Kernel (modelo Transaction compartido)
  • Customer Management ↔ Core Banking: Anti-Corruption Layer (CRM legacy con adapter)
  • Infrastructure ↔ todos: Open Host Service (APIs genéricas)

S8: Puntos Críticos de Fallo (Top 3)

FP-001: Proveedor KYC Externo No Disponible

Consecuencia:
Onboarding detenido. Pérdida de clientes potenciales.
Probabilidad:
Media (2-3 incidentes/mes)
Impacto:
Alto (bloquea adquisición)
MTTR actual:
2-4 horas

Mejora: Segundo proveedor KYC como fallback + cache de verificaciones (TTL 24h). MTTR → <5 min.

FP-002: Race Condition en Payment Engine (Doble Débito)

Consecuencia:
Pérdida financiera directa. Riesgo regulatorio.
Probabilidad:
Baja (idempotency keys actuales)
Impacto:
Crítico (monetario + confianza)
MTTR actual:
12h (detección batch)

Mejora: Distributed lock con Redis + detección real-time vía event sourcing. MTTR → <1 min.

FP-003: SWIFT Gateway Timeout en Pagos Internacionales

Consecuencia:
Pagos internacionales retenidos. Incumplimiento SLA corporativo.
Probabilidad:
Media (ventanas de mantenimiento SWIFT)
Impacto:
Alto (reputacional + contractual)
MTTR actual:
Variable (depende de SWIFT)

Mejora: Ruta alternativa vía corresponsal para montos <$50K. Notificación proactiva con ETA.
