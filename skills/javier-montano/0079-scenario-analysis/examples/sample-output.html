Proyecto: Acme Corp | Fecha: 12 de marzo de 2026 | Variante: Técnica | 3 escenarios evaluados

1. Contexto del Análisis

Acme Corp opera una plataforma core banking basada en COBOL/Java monolítico desplegada on-premise desde 2008. El sistema procesa 2.4M transacciones/día con 99.7% uptime pero enfrenta: costos de mantenimiento crecientes (+18% YoY), incapacidad de lanzar productos digitales en menos de 6 meses, y deuda técnica que limita la integración con fintechs.

Aunque el Escenario A tiene score numérico superior (7.40), su limitación estratégica (4/10) lo convierte en una solución temporal. El Escenario B ofrece el mejor balance: riesgo gestionable, costo justificable ($2.2M Yr1 con ROI en Yr2), y habilitación de 3+ objetivos estratégicos.

9. Implementation Roadmap — Escenario B

Phase 1: Foundation (Meses 1-2)

DDD Event Storming workshops. Bounded context mapping. Cloud landing zone. Team: 3 existing + 2 new hires.

Gate: Domain model validated + cloud sandbox operational.

Phase 2: Pilot (Meses 3-5)

Payments microservice development. Anti-corruption layer. CI/CD + observability. Integration testing.

Gate: 10K tx/day with <200ms p99 latency.

Phase 3: Scale (Meses 6-10)

Accounts + Lending microservices. Parallel run 8 weeks. Data migration. Load testing 2.4M+ tx/day.

Gate: Zero critical incidents during 4-week parallel run.

Phase 4: Optimize (Meses 10-12)

Legacy decommissioning. Cost optimization. Performance tuning. Compliance audit.

Gate: Ops/Finance/Security/Compliance sign-off.
