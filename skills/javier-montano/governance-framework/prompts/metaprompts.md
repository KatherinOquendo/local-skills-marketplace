# Metaprompts — Governance Framework

## Metaprompt 1: Governance Proportionality Check

**Strategy**: Ensure governance intensity matches project risk, size, and strategic importance.

**Protocol**:
1. Classify the project on a 3-tier scale (Tier 1: Strategic, Tier 2: Tactical, Tier 3: Operational)
2. Map current governance controls to each tier
3. Identify over-governed areas (unnecessary overhead) and under-governed areas (risk exposure)
4. Recommend right-sized governance for each tier
5. Tag adjustments with [PLAN] and rationale

**Activation**:
> "Review this governance framework for proportionality. Is the governance intensity appropriate for each project tier? Identify where we are over-governing (adding cost without value) and under-governing (accepting risk without awareness)."

## Metaprompt 2: Decision Bottleneck Analysis

**Strategy**: Identify and resolve decision-making bottlenecks in the governance structure.

**Protocol**:
1. Map all decision types and their approval paths
2. Calculate average decision cycle time per type
3. Identify bottleneck nodes (people, committees, or processes)
4. Propose parallel approval paths or delegation authority
5. Simulate improved cycle times with [INFERENCIA] tags

**Activation**:
> "Analyze this governance framework for decision bottlenecks. Map every decision type to its approval path, estimate cycle times, and identify where decisions stall. Propose structural changes to reduce decision latency by at least 40%."

## Metaprompt 3: Governance Failure Mode Analysis

**Strategy**: Apply FMEA thinking to governance structures to prevent systemic failures.

**Protocol**:
1. List all governance mechanisms (gates, reviews, approvals)
2. For each, identify failure modes (bypassed, ignored, rubber-stamped)
3. Assess severity, probability, and detectability
4. Calculate Risk Priority Number (RPN)
5. Propose preventive controls for top-5 failure modes

**Activation**:
> "Perform a Governance FMEA on this framework. For each governance mechanism, identify how it could fail, the impact of failure, and how likely we would detect the failure before consequences materialize. Prioritize improvements by Risk Priority Number."
