# Canonical Skill Numbering System (0000-9999)

## Principle: User Proximity

Skills are ordered from **closest to the user** (input/interaction) to **furthest from the user** (infrastructure/meta). The user is the center of gravity.

## Ranges

| Range | Category | Proximity | Description |
|-------|----------|-----------|-------------|
| **0000-0999** | Input & Analysis | Nearest | User input processing, requirements, discovery |
| **1000-1999** | Design & Architecture | Near | System design, component design, UX |
| **2000-2999** | Frontend & UI | Near-Mid | What the user sees and touches |
| **3000-3999** | Backend & Logic | Mid | Business logic, APIs, services |
| **4000-4999** | Data & Storage | Mid-Far | Databases, schemas, migrations |
| **5000-5999** | Security & Auth | Mid-Far | Authentication, authorization, compliance |
| **6000-6999** | Testing & Quality | Far | Verification, validation, QA |
| **7000-7999** | DevOps & Deployment | Far | CI/CD, hosting, environments |
| **8000-8999** | Performance & Optimization | Far | Speed, SEO, bundles, caching |
| **9000-9999** | Meta & Orchestration | Furthest | Skills about skills, session mgmt, tooling |
