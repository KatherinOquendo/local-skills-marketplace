Cómo Funciona la Auto-Selección

El ecosistema infiere intención desde señales lingüísticas en 0–2 inputs y activa el stack óptimo de skills, agentes y orquestación.

0 inputs — Bootstrap

El SessionStart hook escanea el directorio, lee session-state.json y presenta un Ghost Menu con las 3 acciones más probables. El usuario solo necesita elegir.

1 input — Routing

El Intent Router detecta el dominio (Discovery / Architecture / Security / PM) y selecciona 3–5 skills canónicos en orden óptimo. Modo Express por defecto.

2 inputs — Personalización

Con contexto + restricciones, el Agent Committee Spawner activa agentes especializados y escala al modo Guided o Deep según complejidad detectada.

Progressive Loading

Los skills se cargan en L1 (metadata) para routing. Solo los seleccionados escalan a L2 (core) o L3 (deep). Máximo 5 skills activos simultáneamente.

Evidence-First

Todo claim se etiqueta automáticamente [EXPLICIT/INFERRED/OPEN]. Si >30% son supuestos, el sistema avisa antes de entregar. Zero hallucination por diseño.

Continuidad

Session-state.json + changelog garantizan continuidad entre conversaciones. El sistema recuerda la fase, las decisiones y los supuestos pendientes.
