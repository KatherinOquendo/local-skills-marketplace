# UX Writing — Use Case Prompts

## Prompt 1 — Architecture Document Readability Overhaul

> Audit this 40-page architecture document for readability and accessibility.
> Restructure the information hierarchy, add scanability elements, and reduce
> average sentence length to under 20 words. Preserve all technical content
> while making it accessible to both architects and engineering managers.

## Prompt 2 — API Documentation UX Improvement

> Improve the UX of this API reference documentation. Add navigation aids,
> restructure for random access (developers look up specific endpoints),
> and ensure code examples are scannable. Apply the Diataxis framework to
> separate tutorials from reference content.

## Prompt 3 — Incident Report Readability

> Rewrite this post-incident report for dual-audience readability: engineering
> managers (want impact and timeline) and senior engineers (want root cause
> and remediation details). Apply progressive disclosure so each audience
> can find their content within 30 seconds.
