# Roadmap PoC — Use Case Prompts

## Prompt 1 — Event-Driven Architecture PoC

> Design a PoC plan to validate whether Kafka-based event sourcing can replace
> the current synchronous REST architecture for order processing. Define hypotheses
> for throughput, latency, and operational complexity. Plan 3 sprints with gates.
> Team: 2 senior engineers, 1 DevOps. Budget: 6 FTE-weeks.

## Prompt 2 — AI Document Processing PoC

> Plan a proof of concept for automated document classification and extraction
> using LLMs. Hypotheses: accuracy >95% for 5 document types, processing time
> <30 seconds per document, cost <$0.10 per document. Define sprints, success
> criteria, and the go/no-go framework for production investment.

## Prompt 3 — Database Migration Validation

> Create a PoC roadmap to validate migrating from MongoDB to PostgreSQL for
> a real-time analytics workload. Test hypotheses around query performance,
> schema flexibility, and operational tooling. Include a risk register for
> data migration and a rollback plan for each sprint.
