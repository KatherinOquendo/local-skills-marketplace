# Observability Architecture — Use Case Prompts

## Prompt 1 — Observability Stack Design
> "Design the observability architecture for this microservices system. Specify logging format, tracing instrumentation, metrics collection (RED/USE), correlation strategy, and backend selection. Include OpenTelemetry Collector configuration and Grafana dashboard specifications."

## Prompt 2 — SLO Framework
> "Define SLOs for this application's critical user journeys. Identify SLIs, set targets, calculate error budgets, and design burn rate alerting. Produce SLO dashboard specifications and error budget policy (what happens when budget is exhausted)."

## Prompt 3 — Incident Response Design
> "Design the incident response process for this engineering team. Define severity levels, on-call rotation, escalation paths, runbook templates, postmortem process, and incident metrics. Include automation for common incident types."
