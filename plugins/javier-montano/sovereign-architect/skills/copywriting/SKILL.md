# Copywriting

Produces persuasive technical copy — value propositions, calls to action, and executive
summaries — that translates engineering capabilities into language that drives stakeholder
decisions and actions.

## Guiding Principle

> *"Good technical copy does not describe features. It describes the future state the reader wants and the path to get there."*

## Procedure

### Step 1 — Audience and Action Analysis

1. Define who will read this copy and what action they should take.
2. Identify the reader's current beliefs, concerns, and decision criteria.
3. Determine the emotional and rational triggers that drive this audience.
4. Establish the one thing the reader must remember if they forget everything else.

### Step 2 — Value Proposition Design

1. Articulate the core value proposition: what changes for the reader and why it matters.
2. Frame benefits in terms the audience uses (business outcomes, not technical features).
3. Create contrast between the current state (pain) and the future state (gain).
4. Support claims with specific, quantified evidence where available.

### Step 3 — Copy Production

1. Write headlines that convey the single most important message.
2. Structure body copy with the inverted pyramid: conclusion first, then supporting detail.
3. End each section with a clear call to action or next step.
4. Use active voice, concrete nouns, and strong verbs.

### Step 4 — Copy Refinement

1. Cut every word that does not serve the reader's understanding or motivation.
2. Verify that technical accuracy is preserved in simplified language.
3. Test the copy against the target action: does it make the reader want to act?
4. Ensure consistent voice and tone with the organizational brand.

## Quality Criteria

- Value proposition is articulated in one sentence that a non-technical reader can understand.
- Every section ends with a clear call to action or next step.
- Technical accuracy is preserved — simplification does not introduce inaccuracy.
- Copy is concise: no section exceeds the minimum length needed to make its point.

## Anti-Patterns

- Leading with technology features instead of reader benefits.
- Using superlatives ("best," "fastest," "most advanced") without evidence.
- Writing calls to action that are vague ("learn more") instead of specific ("schedule a 30-minute architecture review").
- Producing copy that requires prior technical knowledge to understand.
