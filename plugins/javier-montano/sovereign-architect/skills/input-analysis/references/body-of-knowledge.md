# Input Analysis — Body of Knowledge

## Core Concepts

1. **Intent Amplification**: The practice of extracting maximum value from an input by interpreting it through multiple lenses (literal, contextual, strategic) to surface implicit needs.
2. **Ambiguity Resolution**: Identifying terms, requirements, or statements that admit multiple interpretations and proposing specific resolutions with confidence levels.
3. **Contradiction Detection**: Finding statements within or across inputs that cannot both be true simultaneously. Contradictions are high-priority resolution items.
4. **Source Reliability**: Assessing the trustworthiness and currency of input sources. First-hand technical documentation ranks higher than secondhand summaries.
5. **Unstated Assumption**: An implicit belief that the input relies upon but does not make explicit. Surfacing assumptions prevents downstream analysis from inheriting hidden biases.
6. **Signal-to-Noise Ratio**: The proportion of actionable information vs filler, boilerplate, or irrelevant content in an input. High-noise inputs require more aggressive filtering.

## Patterns

1. **Three-Lens Interpretation**: Reading every input through literal (what it says), contextual (what it means given the situation), and strategic (what it implies for the broader objective) lenses.
2. **Assumption Extraction Protocol**: For every requirement, ask "what must be true for this to make sense?" and document each answer as an explicit assumption.
3. **Gap Analysis Matrix**: Cross-referencing expected input types against actual inputs to identify what is missing before analysis begins.
4. **Confidence Scoring**: Assigning each interpreted requirement a confidence level (High/Medium/Low) based on evidence quality and ambiguity.

## Tools

1. **Requirements Parsing Templates**: Structured templates for decomposing natural language requirements into atomic, testable statements.
2. **Contradiction Detection Checklists**: Systematic questions to surface inconsistencies across input documents.
3. **Mind Mapping**: Visual brainstorming tool for exploring intent amplification and surfacing adjacent concerns.
4. **Affinity Diagramming**: Grouping related inputs to identify themes, gaps, and contradictions.
