## Slide 2: Client Value Map (Hero KPIs)

**Layout:** 4-card grid on white background, each card dark with gold number

**Content:**
- **4 Hero Value Cards:**
  - Magnitude: Alto — Ahorro operativo anual (automatización de procesos manuales)
  - Magnitude: Medio-Alto — Revenue enablement (nuevos canales digitales, 12-24 meses)
  - Magnitude: Alto — Risk reduction (compliance regulatorio, evita sanciones)
  - Magnitude: Medio — Time-to-market (reducción de lead time de features, 50-70%)
- **Narrative:** "Cuatro fuentes de valor cuantificables que fundamentan la estructura comercial propuesta."
- **Footnote:** "Magnitudes basadas en análisis de discovery. Cuantificación precisa requiere baseline en Sprint 0."

**Speaker Notes:**
"Antes de hablar de estructura comercial, necesitamos entender QUÉ valor genera esta transformación. Identificamos cuatro fuentes principales. Cada una es medible y tiene un horizonte definido. Esto es la base sobre la cual construimos el modelo."

**Visual Direction:** High-impact data visualization. Cards with gold accent for magnitude. No dollar amounts — use descriptive magnitudes (Alto, Medio-Alto, etc.).

## Slide 4: Pricing Tiers — Engagement Structure

**Layout:** Stacked horizontal bars showing phases, each with fixed + variable components

**Content:**
- **Phase 1 — Foundation (Months 1-6):**
  - Fixed component: Sprint-based delivery, scope defined per sprint
  - Variable component: Bonus tied to deployment velocity KPI
  - Gate: Go/No-Go for Phase 2 based on 3 measurable criteria
- **Phase 2 — Scale (Months 7-12):**
  - Fixed component: Reduced base (team optimization)
  - Variable component: Expanded to include business KPIs (adoption rate, error reduction)
  - Gate: Value review — measured savings vs. projected
- **Phase 3 — Optimize (Months 13-18):**
  - Fixed component: Minimal (support + optimization)
  - Variable component: Revenue share on new capabilities enabled
  - Gate: Transition to BAU model

**Speaker Notes:**
"El modelo evoluciona por fase. En Phase 1, el componente fijo es mayor — estamos construyendo la base. En Phase 2, la proporción variable crece porque ya tenemos métricas. En Phase 3, el modelo se invierte: mínimo fijo, máximo variable. Cada fase tiene un gate de salida — Acme puede renegociar o salir en cada gate."

**Visual Direction:** Stacked bar chart showing fixed (dark) vs. variable (gold) proportions per phase. Visual shows variable growing over time.

## Slide 5: Market Positioning — Competitive Context

**Layout:** 2x2 quadrant chart

**Content:**
- **Axes:** X = Value Alignment (low → high), Y = Predictability (low → high)
- **Quadrants:**
  - Top-Left: "Traditional T&M" — high predictability, low alignment
  - Top-Right: "Our Proposal (Hybrid)" — high predictability, high alignment
  - Bottom-Left: "Ad-hoc / Staff Aug" — low on both
  - Bottom-Right: "Pure Outcome-Based" — high alignment, low predictability
- **Positioning statement:** "Nuestra propuesta maximiza alineamiento de valor SIN sacrificar predictibilidad presupuestaria."

**Speaker Notes:**
"En el mercado, la mayoría de proveedores ofrecen T&M — es simple pero no alinea incentivos. Algunos ofrecen outcome-based puro — riesgoso para ambos sin métricas maduras. Nuestra propuesta se posiciona en el cuadrante superior derecho: máximo alineamiento con predictibilidad para el CFO."

**Visual Direction:** Clean quadrant chart with MetodologIA orange dot in top-right quadrant. Competitors as gray dots in other quadrants. Labels for each quadrant.

**Visual Direction:** Table with color-coded cells (gold for positive, neutral for investment). Waterfall chart showing cumulative value position. CRITICAL: No dollar amounts anywhere on this slide.

## Slide 7: Partner Model & Go-to-Market

**Layout:** Flow diagram showing relationship structure

**Content:**
- **Partnership Structure:**
  - MetodologIA: Technology delivery, architecture, quality assurance
  - Client: Domain expertise, stakeholder access, change management
  - Optional: Third-party specialist (data, security, industry-specific)
- **Governance:**
  - Steering committee: Monthly (CFO, CTO, Delivery Lead)
  - Technical review: Bi-weekly (architects, tech leads)
  - KPI review: Quarterly (tied to variable component)
- **IP Model:**
  - Custom code: Client ownership
  - Reusable frameworks: Shared license (MetodologIA retains, client gets perpetual use)
  - Third-party: Pass-through licensing

**Speaker Notes:**
"La estructura de partnership define quién hace qué y cómo se gobierna la relación. Tres niveles de governance aseguran alineamiento. El modelo de IP es transparente: lo que es custom de Acme es de Acme. Lo que es framework reutilizable se comparte con licencia perpetua para Acme."

**Visual Direction:** Flow diagram with three columns (MetodologIA / Joint / Client). Governance cadence as timeline below. IP model as simple table.

Week 3-4, legal revisa la estructura y negociamos términos. En paralelo, el equipo técnico arranca Sprint 0 para establecer baselines."

**Visual Direction:** Clean horizontal timeline with gold milestone markers. Decision points as dark cards above the timeline. Clear CTA in gold-bordered box at bottom.
