# Output Engineering — Body of Knowledge

## Core Concepts
- **Progressive Disclosure**: Layering information from summary to detail so readers choose their depth.
- **Ghost Menu**: A contextual command palette embedded in artifacts suggesting actionable next steps.
- **TL;DR (Too Long; Didn't Read)**: A standalone summary paragraph that captures the essential conclusion or recommendation.
- **Information Architecture**: The structural design of information to support findability, usability, and comprehension.
- **Markdown-Excellence**: A formatting standard that combines clear hierarchy, data tables, diagrams, and consistent styling.
- **Cognitive Load Reduction**: Designing documents so the reader's working memory is not overloaded — chunking, whitespace, visual hierarchy.
- **Artifact Consistency**: All deliverables in a set follow the same template, tone, and formatting rules.

## Patterns & Practices
- **Inverted Pyramid**: Start with the most important information (conclusion), then supporting details, then background.
- **Scannable Structure**: Use headings, bullets, bold key terms, and tables so readers can skim before deep reading.
- **Consistent Heading Hierarchy**: H1 (title) → H2 (major sections) → H3 (subsections) — never skip levels.
- **Data Tables Over Prose**: When comparing options or listing items, tables are faster to read than paragraphs.
- **Diagram-First**: Lead complex explanations with a diagram, then provide prose that narrates the diagram.
- **Metadata Block**: Every artifact starts with: title, author, date, version, status, audience.

## Tools & Technologies
- **Markdown Renderers**: GitHub, GitLab, Obsidian, Notion, VS Code preview.
- **Mermaid.js**: Inline diagrams rendered from text in Markdown.
- **Linting**: markdownlint for consistent formatting rules.
- **Templates**: Reusable Markdown templates for each artifact type.
- **PDF Generation**: Pandoc, md-to-pdf for polished final deliverables.

- Don Norman, "The Design of Everyday Things" — Usability principles applied to documents.
- Nielsen Norman Group — "F-Pattern Reading" and "Inverted Pyramid" for web content.
- GOV.UK Content Design — Plain language and progressive disclosure standards.
