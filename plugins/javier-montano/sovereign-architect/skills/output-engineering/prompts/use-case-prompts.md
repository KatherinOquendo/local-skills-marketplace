# Output Engineering — Use Case Prompts

## Prompt 1: Format a Technical Discovery
> I have raw notes from a 3-hour technical discovery session covering: current architecture, pain points, team capabilities, and recommended solutions. Transform these notes into a polished discovery report with: TL;DR, key findings with evidence tags, detailed analysis sections, architecture diagrams, risk matrix, and a ghost menu with next steps. Target audience: CTO and engineering leads.

## Prompt 2: Create a Proposal Template
> Design a reusable markdown template for technical proposals that includes: metadata block, executive summary, problem statement, proposed solution, alternatives considered, implementation plan, risk assessment, timeline, team composition, and investment estimate. Include formatting guidelines, ghost menu structure, and evidence tagging requirements.

## Prompt 3: Restructure a Wall of Text
> This 2000-word technical analysis was written as a continuous narrative with no headings, no bullets, and no structure. Restructure it using progressive disclosure: extract the TL;DR, organize into logical sections with headings, convert comparisons into tables, add a diagram where appropriate, and append a ghost menu with suggested follow-up actions. Do not change the content, only the structure and formatting.
