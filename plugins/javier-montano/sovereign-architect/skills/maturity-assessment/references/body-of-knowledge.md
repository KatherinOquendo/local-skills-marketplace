# Body of Knowledge: Maturity Assessment

## Core Concepts

### What This Skill Does
Capability maturity evaluation using CMMI, DORA, and custom models for software engineering practices. This skill provides structured guidance for assess maturity with evidence-based recommendations.

### When to Use
- When evaluating current state of practices in this domain
- When planning improvement initiatives
- When making strategic decisions that require data-driven assessment

### Key Decision Framework

| Factor | Low Maturity | Medium Maturity | High Maturity |
|--------|-------------|-----------------|---------------|
| Process | Ad-hoc, inconsistent | Documented, partially followed | Optimized, continuously improved |
| Tooling | Manual, fragmented | Semi-automated, integrated | Fully automated, self-healing |
| Culture | Reactive, blame-focused | Proactive, learning-oriented | Generative, high-trust |

## Best Practices

1. Always start with evidence from the existing codebase [HECHO]
2. Use established frameworks for structured assessment [HECHO]
3. Benchmark against industry standards where available [INFERENCIA]
4. Focus on outcomes, not just process adherence [HECHO]

- Consult industry frameworks (CMMI, DORA, ISO, OWASP) for benchmarks
