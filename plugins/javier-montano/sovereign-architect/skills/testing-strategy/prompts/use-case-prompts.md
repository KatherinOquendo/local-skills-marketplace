# Testing Strategy — Use Case Prompts

## Prompt 1 — Testing Strategy Design
> "Design a comprehensive testing strategy for this application. Analyze the current test suite, propose target pyramid ratios, define testing frameworks per layer, design test data management, and specify CI integration. Include coverage targets and quality metrics."

## Prompt 2 — Test Suite Health Audit
> "Audit the test suite of this codebase. Measure test distribution (unit/integration/E2E), identify flaky tests, check coverage gaps on critical paths, evaluate test execution time, and assess test quality (readability, isolation, determinism). Produce a health report."

## Prompt 3 — Contract Testing Implementation
> "Design a contract testing strategy for this microservices system. Identify all service-to-service contracts, select consumer-driven vs. provider-driven approach per integration, design the Pact broker workflow, and integrate into the CI/CD pipeline."
