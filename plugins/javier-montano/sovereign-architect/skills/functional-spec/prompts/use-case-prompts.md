# Functional Specification — Use Case Prompts

## Prompt 1 — User Registration and Onboarding

> Write the functional specification for user registration and onboarding.
> Include all use cases (email signup, SSO, invitation-based), business rules
> for validation and fraud prevention, and a complexity/risk matrix.
> Produce acceptance criteria in Given/When/Then format for each flow.

## Prompt 2 — Multi-Tenant Access Control

> Specify the functional requirements for a multi-tenant access control system.
> Define roles, permissions, and tenant isolation rules. Model use cases for
> role assignment, permission inheritance, and cross-tenant data access.
> Include the traceability matrix from business rules to acceptance criteria.

## Prompt 3 — Payment Reconciliation Engine

> Produce a functional specification for automated payment reconciliation.
> Cover matching rules, exception handling, manual override workflows,
> and audit trail requirements. Score each use case on complexity and risk,
> and identify which items require proof-of-concept validation.
