# Discovery Handover — Use Case Prompts

## Prompt 1 — Microservices Migration Handover

> Produce a handover package for transitioning a microservices migration from
> discovery to implementation. Include ADRs for service boundary decisions,
> the assumption register for performance targets, and implementation-ready
> work packages for the first two sprints.

## Prompt 2 — Platform Team Establishment

> Create the handover document for standing up a new platform engineering team.
> Package the technology selection decisions, team topology recommendations,
> initial backlog with acceptance criteria, and the support model for the
> transition period.

## Prompt 3 — Vendor Integration Handover

> Build a handover package for a third-party payment gateway integration.
> Document API contract decisions, security requirements, fallback strategies,
> and the assumption register for vendor SLA commitments. Include a "start here"
> guide for the implementation team.
