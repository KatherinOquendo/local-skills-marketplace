# Solution Roadmap — Body of Knowledge

## Core Concepts

1. **Investment Horizon**: A time-bounded planning frame (H1: 0-6mo, H2: 6-12mo, H3: 12+mo) with decreasing specificity and increasing optionality as the horizon extends.
2. **Critical Path**: The longest chain of dependent activities that determines the minimum project duration. Delays on the critical path delay the entire roadmap.
3. **Three-Point Estimation**: Using optimistic, most likely, and pessimistic estimates to model effort uncertainty and calculate expected duration with confidence intervals.
4. **Value Stream Alignment**: Ensuring every roadmap phase connects to a measurable improvement in the flow of value from concept to customer.
5. **Phase Gate**: A decision point between phases where stakeholders evaluate progress, validate assumptions, and decide whether to proceed, pivot, or stop.
6. **Dependency Classification**: Hard dependencies (must complete before starting), soft dependencies (preferred but not required), and external dependencies (outside team control).

## Patterns

1. **Walking Skeleton First**: Start with an end-to-end thin slice before expanding. The first phase should prove the architecture works.
2. **Dual-Track Delivery**: Running discovery for Phase N+1 in parallel with delivery of Phase N. Prevents idle time between phases.
3. **Risk-First Scheduling**: Place the highest-risk initiatives earliest in the roadmap to fail fast and preserve option value.
4. **Incremental Value Delivery**: Every phase includes at least one user-facing value delivery, preventing the "infrastructure-only sprint" pattern.

## Tools

1. **Mermaid Gantt Charts**: Timeline visualization with phases, milestones, and dependencies.
2. **PERT Charts**: Network diagrams showing critical path and slack for each activity.
3. **Wardley Maps**: Strategic positioning of roadmap components along the evolution axis.
4. **Now/Next/Later Framework**: Lightweight roadmap format that communicates priority without committing to dates.
