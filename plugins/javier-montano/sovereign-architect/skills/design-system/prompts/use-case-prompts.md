# Design System — Use Case Prompts

## Prompt 1 — Technical Deliverable Design System

> Define a complete design token system for technical deliverables (assessments,
> roadmaps, executive summaries). Include color palette, typography scale,
> spacing tokens, and component definitions for tables, diagrams, code blocks,
> and callout boxes. Produce the canonical token reference file.

## Prompt 2 — HTML Report Template System

> Design the component library and page templates for HTML-based technical
> reports. Include responsive layout tokens, print-friendly adaptations,
> and Mermaid diagram styling. Define the three standard page templates:
> executive summary, detailed assessment, and technical appendix.

## Prompt 3 — Multi-Brand Token Architecture

> Create a design token architecture that supports multiple brands
> (engineering, consulting, product) sharing the same component library
> but with different visual identities. Define the token hierarchy
> (global > brand > component) and the theme switching mechanism.
