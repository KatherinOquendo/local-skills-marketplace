# Cost Estimation — Body of Knowledge

## Core Concepts
- **Work Breakdown Structure (WBS)**: Hierarchical decomposition of project scope into estimable work packages
- **Effort Inductors**: Quantifiable characteristics that drive effort: entities, screens, integrations, endpoints
- **Three-Point Estimation (PERT)**: Optimistic + 4x Most Likely + Pessimistic / 6 for expected duration
- **Cone of Uncertainty**: Estimation accuracy improves as the project progresses: early estimates range 4x, late estimates 1.25x
- **Velocity Calibration**: Using historical team velocity to convert story points to calendar time
- **Risk-Adjusted Estimates**: Applying multipliers to account for uncertainty, complexity, and external dependencies
- **Non-Functional Overhead**: Testing (30-40%), infrastructure (10-15%), documentation (5-10%) as percentage of development effort

## Patterns
- **T-Shirt Sizing**: Quick relative sizing (XS, S, M, L, XL) for early-stage estimation
- **Planning Poker**: Team-based consensus estimation using modified Fibonacci
- **Reference Class Forecasting**: Using outcomes of similar past projects to calibrate estimates
- **Monte Carlo Simulation**: Statistical modeling of estimate ranges to produce probability distributions
- **Affinity Mapping**: Grouping similar items and estimating by analogy to known items

## Tools & Frameworks
- **Jira / Linear**: Work item tracking with velocity and burndown analytics
- **Monte Carlo tools**: Statistical estimation calculators for project forecasting
- **COCOMO II**: Constructive Cost Model for software effort estimation
- **Function Point Analysis**: ISO-standardized method for sizing software functionality
- **Spreadsheet models**: Custom estimation models with sensitivity analysis
