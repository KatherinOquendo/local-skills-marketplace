# Metaprompts — Risk Response Planning

## Meta-Strategy 1: Secondary Risk Detection

**Purpose:** Ensure risk responses don't create new risks

```
For each planned risk response, ask:
1. Does this response create a NEW risk? (e.g., outsourcing creates vendor dependency risk)
2. Does this response increase another existing risk? (e.g., fast-tracking increases quality risk)
3. Is the response cost proportionate to the risk exposure?
4. Does the response have unintended consequences for team morale or stakeholder relationships?
Document all secondary risks identified and add them to the risk register.
```

## Meta-Strategy 2: Response Completeness Audit

**Purpose:** Verify every significant risk has an adequate response

```
Audit response coverage:
1. Every HIGH severity risk must have a specific, actioned response (not just "accept")
2. Every response must have a single OWNER (not "the team")
3. Every response must have a DEADLINE (not "ongoing")
4. Every mitigate/avoid response must have a CONTINGENCY fallback
5. Every transfer response must have contractual documentation
Flag any gap and propose remediation.
```

## Meta-Strategy 3: Response Prioritization under Constraint

**Purpose:** Optimize response investment when budget is limited

```
When response budget is constrained:
1. Rank responses by risk exposure reduced per FTE-month invested
2. Identify responses with synergy (one action mitigates multiple risks)
3. Identify low-cost responses with high impact (quick wins)
4. Identify risks where "accept" is genuinely appropriate (not just default)
Produce a prioritized response implementation roadmap within budget constraints.
```
