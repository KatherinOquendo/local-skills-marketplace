# Body of Knowledge — Project Selection

## 1. Definition and Scope

Project selection is the process of evaluating, comparing, and choosing projects for investment based on strategic alignment, resource availability, risk profile, and expected value. It is a portfolio-level decision that determines which initiatives receive organizational commitment. [DOC]

## 3. Selection Methods

### 3.1 Financial Methods
- Net Present Value (NPV)
- Internal Rate of Return (IRR)
- Payback Period
- Return on Investment (ROI)
- Cost-Benefit Analysis [METRIC]

### 3.2 Strategic Methods
- Strategic alignment scoring
- Balanced scorecard linkage
- OKR contribution mapping
- Portfolio balance analysis (risk vs. reward matrix)
- Innovation vs. maintenance allocation [PLAN]

### 3.3 Scoring Models
- Weighted scoring matrix
- Analytic Hierarchy Process (AHP)
- TOPSIS (Technique for Order Preference)
- Pairwise comparison
- Forced ranking [METRIC]

## 4. Selection Criteria Categories

| Category | Example Criteria | Weight Range |
|----------|-----------------|-------------|
| Strategic fit | Alignment to corporate strategy | 20-30% |
| Financial value | NPV, ROI, payback period | 20-30% |
| Risk profile | Technical, market, execution risk | 10-20% |
| Resource feasibility | Skills availability, capacity | 10-15% |
| Urgency | Regulatory deadline, competitive threat | 5-15% |
| Synergy | Dependencies with other initiatives | 5-10% |

## 5. Portfolio Balancing

Projects should be selected to achieve a balanced portfolio across:
- Risk levels (high/medium/low)
- Time horizons (short/medium/long)
- Innovation types (core/adjacent/transformational — 70/20/10 rule)
- Business units and strategic themes
- Resource types and skill requirements [PLAN]

## 6. Decision Governance

| Gate | Decision | Authority | Input |
|------|----------|-----------|-------|
| Idea screening | Advance to analysis | Portfolio analyst | Problem statement |
| Business case review | Advance to selection | Investment committee | Full business case |
| Selection decision | Fund or reject | Portfolio board | Selection scorecard |
| Periodic rebalancing | Continue, pivot, kill | Portfolio board | Health check data |

## 7. Common Pitfalls

| Pitfall | Impact | Mitigation |
|---------|--------|-----------|
| HiPPO effect (highest paid person's opinion) | Biased selection | Structured scoring + blind ranking |
| Sunk cost fallacy | Continuing failing projects | Kill criteria + periodic review |
| Pet project syndrome | Non-strategic projects funded | Mandatory strategic alignment score |
| Analysis paralysis | Selection delayed indefinitely | Time-boxed decision process |
| Survivorship bias | Only seeing successful project types | Include failure analysis in criteria |
