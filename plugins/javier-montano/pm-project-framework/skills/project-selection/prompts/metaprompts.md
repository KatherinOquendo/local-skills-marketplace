# Metaprompts — Project Selection

## Meta-Strategy 1: Selection Bias Detection

**Purpose:** Identify cognitive biases in the selection process

```
Before finalizing any project selection, check for:
1. ANCHORING — Is the first proposal evaluated setting the bar for all others?
2. CONFIRMATION BIAS — Are evaluators seeking data that supports their preferred project?
3. GROUPTHINK — Did the committee converge too quickly without dissent?
4. AVAILABILITY BIAS — Are recent successes/failures disproportionately influencing scores?
5. SUNK COST — Are existing investments biasing continuation decisions?
For each bias detected, apply a correction factor and re-score. Document adjustments.
```

## Meta-Strategy 2: Sensitivity Analysis Protocol

**Purpose:** Test robustness of selection decisions to criteria weight changes

```
For the current selection ranking, test sensitivity:
1. Vary each criterion weight by +/- 20% and recalculate rankings
2. Identify which criteria changes cause ranking reversals
3. For top 2 ranked projects, determine the minimum weight change that would change the selection
4. Flag any selection decision that reverses with < 10% weight change as FRAGILE
Present results in a sensitivity matrix with confidence assessment.
```

## Meta-Strategy 3: Contrarian Portfolio Test

**Purpose:** Challenge portfolio composition through devil's advocacy

```
After selection, argue against the chosen portfolio:
1. What strategic risks does this portfolio NOT address?
2. Which rejected projects might outperform selected ones if market conditions change by 15%?
3. Is the portfolio over-concentrated in any dimension (technology, market, team)?
4. What happens if the highest-value project fails — does the portfolio still deliver acceptable returns?
Recommend portfolio insurance actions based on findings.
```
