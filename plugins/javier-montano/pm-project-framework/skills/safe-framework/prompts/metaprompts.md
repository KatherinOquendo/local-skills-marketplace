# Metaprompts — SAFe Framework

## Meta-Strategy 1: SAFe Overhead Optimization

**Purpose:** Ensure SAFe ceremony investment delivers proportional value

```
Audit SAFe ceremony ROI:
1. Total ceremony hours per PI per person. If > 25% of working hours, optimize.
2. For each ceremony: what VALUE would be lost if it were cancelled?
3. Are ceremonies following SAFe guidance on timeboxing? Measure actual vs. prescribed.
4. Are ceremony outputs (PI objectives, improvement items) being acted upon?
Reduce or redesign ceremonies that score low on value delivery.
```

## Meta-Strategy 2: SAFe Anti-Pattern Detection

**Purpose:** Identify common SAFe implementation failures

```
Check for these anti-patterns:
1. WATER-SCRUM-FALL — Agile teams but waterfall planning and governance
2. PI PLANNING THEATER — Planning event occurs but plans are not followed
3. FEATURE FACTORY — Teams deliver features without validating business outcomes
4. DEPENDENCY ADDICTION — More dependencies after SAFe than before
5. METRICS GAMING — Teams optimizing for metrics rather than outcomes
For each detected anti-pattern, recommend corrective action.
```

## Meta-Strategy 3: Right-Sizing SAFe Configuration

**Purpose:** Ensure SAFe configuration matches organizational needs

```
Evaluate SAFe configuration fit:
1. Is the organization using MORE SAFe elements than needed? (Over-framework)
2. Is the organization using FEWER elements than needed? (Under-framework)
3. Are all mandatory Essential SAFe elements in place?
4. Are optional elements providing measurable value?
Recommend configuration adjustments: add, remove, or simplify SAFe elements.
```
