# Body of Knowledge — SAFe Framework

## 1. Definition and Scope

The Scaled Agile Framework (SAFe) is a set of organization and workflow patterns for implementing agile practices at enterprise scale. It provides structured guidance for roles, responsibilities, planning, and managing work across multiple agile teams through Agile Release Trains (ARTs). [DOC]

## 2. SAFe Configurations

| Configuration | Teams | Scope | Key Element |
|--------------|:-----:|-------|-------------|
| Essential SAFe | 5-12 | Single ART | ART, PI Planning, Team/Program levels |
| Large Solution | 12-50 | Multiple ARTs | Solution Train, Supplier integration |
| Portfolio SAFe | 50+ | Enterprise | Lean Portfolio Management, strategic themes |
| Full SAFe | 100+ | Complete enterprise | All 4 levels, value streams |

## 3. Core Values

- **Alignment** — Strategic direction through PI Planning and portfolio Kanban
- **Built-in Quality** — TDD, CI/CD, architecture runway
- **Transparency** — Visible work, metrics, and progress
- **Program Execution** — Reliable, predictable delivery [PLAN]

## 4. SAFe Principles

| # | Principle | Application |
|---|-----------|-------------|
| 1 | Take an economic view | WSJF prioritization, cost of delay |
| 2 | Apply systems thinking | Value stream mapping, holistic optimization |
| 3 | Assume variability; preserve options | Set-based design, decision points |
| 4 | Build incrementally with fast feedback | MVPs, sprint demos, PI reviews |
| 5 | Base milestones on objective evaluation | Objective metrics, not opinion |
| 6 | Make value flow without interruptions | Limit WIP, reduce batch sizes |
| 7 | Apply cadence, synchronize with cross-domain planning | PI cadence, ART sync |
| 8 | Unlock intrinsic motivation | Autonomy, mastery, purpose |
| 9 | Decentralize decision-making | Team-level decisions, guardrails |
| 10 | Organize around value | Value streams, not functional silos |

## 5. Key Ceremonies

| Ceremony | Level | Duration | Purpose |
|----------|-------|----------|---------|
| PI Planning | ART | 2 days | Align all teams on next PI objectives |
| Sprint Planning | Team | 2-4 hours | Plan next iteration |
| System Demo | ART | 1 hour | Demonstrate integrated increment |
| Inspect & Adapt | ART | 3-4 hours | Retrospective + improvement planning |
| Scrum of Scrums | ART | 30 min daily | Cross-team coordination |

## 6. Key Roles

| Role | Level | Primary Responsibility |
|------|-------|----------------------|
| Release Train Engineer (RTE) | ART | ART facilitation and coordination [STAKEHOLDER] |
| Product Manager | ART | Program backlog management |
| System Architect | ART | Architecture guidance and runway |
| Scrum Master | Team | Team facilitation and impediment removal |
| Product Owner | Team | Team backlog management |
