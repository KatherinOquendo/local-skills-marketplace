# Metaprompts — Cost Estimation

## Meta-Strategy 1: Anchoring Prevention
**Purpose**: Prevent first estimates from biasing subsequent estimation.
**Strategy**: Use silent/independent estimation before group discussion. Never share the "expected" budget before estimation. Use planning poker for independence.
**Application template**:
> For `{estimation_session}`, ensure independent estimation before discussion. If stakeholder has shared a target number, explicitly note anchoring risk and estimate independently. [INFERENCIA]

## Meta-Strategy 2: Unknown-Unknowns Buffer
**Purpose**: Account for work not yet visible in the WBS.
**Strategy**: Historical data shows 15-25% of project work is not in initial WBS. Apply buffer based on project novelty and definition maturity.
**Application template**:
> For `{estimate}` at Class `{class}` maturity, apply unknown-unknowns buffer: Class 5 = 30%, Class 4 = 20%, Class 3 = 15%, Class 2 = 10%, Class 1 = 5%. Document buffer explicitly. [PLAN]

## Meta-Strategy 3: Estimate vs. Commitment Distinction
**Purpose**: Prevent estimates from becoming commitments prematurely.
**Strategy**: Always present estimates as ranges, not single numbers. Label confidence level. Never allow a P50 estimate to become a budget ceiling without contingency.
**Application template**:
> When presenting `{estimate}`, always show range: P50 (most likely) and P80 (budget). Explicitly state: "This is an estimate at X% confidence, not a commitment." If stakeholder needs a commitment, use P80 with documented assumptions. [STAKEHOLDER]
