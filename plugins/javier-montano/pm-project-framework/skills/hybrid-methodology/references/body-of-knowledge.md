# Body of Knowledge — Hybrid Methodology

## 1. Foundations of Hybrid Methodology

Hybrid methodology is the deliberate combination of predictive and adaptive practices tailored to project context. Unlike ad-hoc mixing, a formal hybrid methodology defines decision criteria, integration points, and governance overlays. [DOC]

### 1.1 Core Principles

| Principle | Description | Source |
|-----------|-------------|--------|
| Context-Driven Selection | Methodology fits project, not vice versa | DA, PRINCE2 Agile |
| Governed Flexibility | Agile within defined guardrails | SAFe, PRINCE2 Agile |
| Flow Optimization | Minimize handoff waste between approaches | Lean, Kanban |
| Continuous Calibration | Adapt the methodology as context evolves | Retrospectives, assessments |
| Evidence-Based Decisions | Data drives methodology changes | Flow metrics, EVM |

### 1.2 Hybrid Patterns

1. **Water-Scrum-Fall** — Predictive planning/closure with Scrum execution. [DOC]
2. **Agile with Gates** — Sprint-based delivery with milestone governance. [DOC]
3. **Phased Agile** — Waterfall phases containing agile iterations. [PLAN]
4. **Kanban Backbone** — Kanban for flow management with Scrum ceremonies. [DOC]
5. **Bimodal** — Separate tracks for stability (predictive) and innovation (agile). [DOC]
6. **Continuous Delivery with Governance** — DevOps flow with compliance checkpoints. [INFERENCIA]

## 2. Standards and Frameworks

### 2.1 PMI Disciplined Agile

DA's goal-driven approach provides 24 process goals with options for hybrid configuration, from initial lifecycle selection through detailed practice choices. [DOC]

### 2.2 PRINCE2 Agile

Integrates PRINCE2's seven principles, themes, and processes with agile delivery concepts. The Agilometer guides the degree of agile adoption per project. [DOC]

### 2.3 ISO 21502:2020

Provides guidance on project management that is methodology-agnostic, enabling hybrid implementations within a standards-compliant framework. [DOC]

## 3. Integration Points

| Predictive Element | Agile Element | Integration Mechanism |
|-------------------|---------------|----------------------|
| Project Charter | Product Vision | Charter references vision/backlog |
| WBS | Product Backlog | WBS maps to epics/features |
| Gantt Chart | Sprint Plan | Gantt shows sprint boundaries |
| Stage Gate | Sprint Review | Gate reviews sprint increment |
| Change Control | Backlog Refinement | CCB for budget; PO for scope detail |
| Status Report | Burndown/Burnup | Combined dashboard |
| Lessons Learned | Retrospective | Quarterly LL from sprint retros |

## 4. Lifecycle Models

### 4.1 Serial Hybrid
Phases: Initiation (Predictive) → Planning (Predictive) → Execution (Agile) → Testing (Agile) → Closure (Predictive)

### 4.2 Parallel Hybrid
Workstreams: Infrastructure (Predictive) || Application (Agile) || Change Mgmt (Adaptive)

### 4.3 Embedded Hybrid
Agile teams operate within a predictive program structure with shared milestones.

## 6. Common Pitfalls

1. **Frankensteining** — Randomly mixing practices without coherent design [INFERENCIA]
2. **Governance Mismatch** — Agile teams under waterfall gates without adaptation
3. **Dual Reporting** — Teams reporting in both agile and predictive formats
4. **Culture Clash** — Predictive managers micromanaging agile teams
5. **No Calibration** — Static hybrid that never adjusts to learning
