# Use-Case Prompts — Hybrid Methodology

## Prompt 1: Design Hybrid Methodology from Assessment

**When**: Assessment results are available and a tailored hybrid approach needs designing.

**Context variables**:
- `{ASSESSMENT_RESULTS}`: Summary of hybrid assessment scores
- `{PROJECT_TYPE}`: Type of project (software, transformation, infrastructure)
- `{TEAM_SIZE}`: Number of team members

**Deliver**:
> "Design a hybrid methodology for a {PROJECT_TYPE} project with {TEAM_SIZE} team members based on these assessment results: {ASSESSMENT_RESULTS}. Define the lifecycle model, ceremony calendar, artifact requirements, integration points between predictive and agile elements, and governance overlay. Include a visual lifecycle diagram."

## Prompt 2: Migrate from Waterfall to Hybrid

**When**: An organization wants to transition from pure waterfall to hybrid delivery.

**Context variables**:
- `{ORG_NAME}`: Organization name
- `{CURRENT_PROCESS}`: Current waterfall process description
- `{PAIN_POINTS}`: Known delivery issues

**Deliver**:
> "Create a hybrid methodology transition plan for {ORG_NAME} moving from {CURRENT_PROCESS}. Current pain points: {PAIN_POINTS}. Design a phased transition (3-6 months) that preserves essential governance while introducing agile practices. Include change management approach, training plan, and success metrics."

## Prompt 3: Scale Hybrid Across Portfolio

**When**: A proven hybrid approach needs scaling from pilot to portfolio level.

**Context variables**:
- `{PILOT_RESULTS}`: Results from hybrid pilot
- `{PORTFOLIO_SIZE}`: Number of projects to scale to
- `{CONSTRAINTS}`: Organizational constraints

**Deliver**:
> "Scale the hybrid methodology from pilot to {PORTFOLIO_SIZE} projects. Pilot results: {PILOT_RESULTS}. Constraints: {CONSTRAINTS}. Define methodology tiers (simple/standard/complex), shared governance framework, center of excellence model, and tooling standardization plan."
