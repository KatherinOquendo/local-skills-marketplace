# Body of Knowledge — Kanban Assessment

## 1. Foundations of Kanban Assessment

Kanban assessment evaluates an organization's or team's implementation of Kanban principles and practices against maturity models. It identifies improvement opportunities in flow management, WIP control, and continuous improvement practices. [DOC]

### 1.1 Kanban Principles (KMM)

| Principle | Description | Assessment Focus |
|-----------|-------------|-----------------|
| Start with what you do now | Respect current processes | Change management approach |
| Agree to pursue incremental change | Evolutionary improvement | Improvement cadence |
| Encourage leadership at all levels | Distributed decision-making | Team empowerment |

### 1.2 Kanban Practices

1. **Visualize the workflow** — Board design quality and accuracy [METRIC]
2. **Limit work in progress** — WIP limits defined and respected [METRIC]
3. **Manage flow** — Cycle time and throughput tracking [METRIC]
4. **Make policies explicit** — Clear, visible process rules [DOC]
5. **Implement feedback loops** — Cadences for review and improvement [PLAN]
6. **Improve collaboratively** — Data-driven experimentation [INFERENCIA]

## 2. Kanban Maturity Model (KMM)

### 2.1 Maturity Levels

| Level | Name | Characteristics |
|-------|------|----------------|
| 0 | Oblivious | No awareness of Kanban principles |
| 1 | Emerging | Basic visualization, no WIP limits |
| 2 | Defined | WIP limits set, basic metrics tracked |
| 3 | Fit-for-Purpose | Service-level expectations defined, data-driven |
| 4 | Risk-Hedged | Probabilistic forecasting, portfolio optimization |
| 5 | Market Leader | Continuous innovation in flow management |

### 2.2 Assessment Dimensions

| Dimension | KMM Focus | Measurement |
|-----------|----------|-------------|
| Visualization | Board accuracy, information radiator quality | Observation |
| WIP Management | Limits defined, respected, optimized | Compliance rate |
| Flow Metrics | Cycle time, throughput, flow efficiency | Data analysis |
| Feedback Loops | Cadence implementation, action follow-through | Cadence audit |
| Policies | Explicitness, visibility, compliance | Policy review |
| Improvement | Experiment rate, outcome tracking | Improvement log |

## 3. Assessment Methods

| Method | Effort | Depth | Best For |
|--------|--------|-------|---------|
| Self-Assessment Survey | Low | Surface | Quick health check |
| Facilitated Workshop | Medium | Moderate | Team awareness building |
| Expert Assessment | High | Deep | Transformation planning |
| Flow Metric Analysis | Medium | Data-driven | Objective measurement |
| Value Stream Mapping | High | Comprehensive | End-to-end optimization |

## 4. Key Metrics for Assessment

| Metric | Formula | Target Range |
|--------|---------|-------------|
| Cycle Time | End Date - Start Date | Stable or decreasing |
| Throughput | Items completed / time period | Stable or increasing |
| WIP Age | Current Date - Start Date | Below SLE threshold |
| Flow Efficiency | Touch Time / Cycle Time | > 15% (typical target) |
| Blocker Rate | Blocked items / total WIP | < 10% |
| WIP Compliance | Days within limit / total days | > 90% |

## 5. Common Pitfalls

1. **Kanban Theater** — Board exists but WIP limits are ignored [INFERENCIA]
2. **Metric Vanity** — Tracking metrics without acting on them
3. **Assessment Once** — Single assessment without continuous monitoring
4. **Ignoring Upstream** — Only assessing development, not demand management
5. **Tool Confusion** — Equating Jira board setup with Kanban maturity
