# Use-Case Prompts — Continuous Improvement

## Prompt 1: Improvement Opportunity Analysis
**When**: Post-retrospective or quarterly review.
**Context variables**: `{retro_actions}`, `{project_metrics}`, `{team_feedback}`
**Deliver**:
> Analyze `{retro_actions}`, `{project_metrics}`, and `{team_feedback}` to identify top improvement opportunities. Prioritize using Impact/Effort matrix. Design experiments for top 3 improvements with hypotheses and measurement criteria.

## Prompt 2: Improvement Impact Report
**When**: Quarterly improvement review.
**Context variables**: `{improvements_implemented}`, `{before_after_metrics}`
**Deliver**:
> Report on impact of `{improvements_implemented}` using `{before_after_metrics}`. Calculate ROI per improvement. Identify which improvements to make permanent and which to abandon.

## Prompt 3: Process Optimization Analysis
**When**: Persistent bottleneck identified.
**Context variables**: `{process_description}`, `{bottleneck_data}`, `{flow_metrics}`
**Deliver**:
> Analyze `{process_description}` for waste and bottlenecks using `{bottleneck_data}` and `{flow_metrics}`. Apply lean thinking to identify elimination, simplification, or automation opportunities. Provide improvement roadmap.
