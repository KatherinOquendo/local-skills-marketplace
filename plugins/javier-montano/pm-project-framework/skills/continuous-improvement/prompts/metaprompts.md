# Metaprompts — Continuous Improvement

## Meta-Strategy 1: Improvement vs. Change Fatigue Balance
**Purpose**: Ensure improvement velocity does not cause change fatigue.
**Strategy**: Limit active improvements to 1-2 per sprint. Complete and stabilize before introducing new changes. Track team sentiment alongside improvement metrics.
**Application template**:
> Review active improvements for `{team}`. If >2 concurrent improvements, defer lower-priority items. Check team satisfaction trend — if declining, pause improvements for 1 sprint. [STAKEHOLDER]

## Meta-Strategy 2: Systemic vs. Local Improvement
**Purpose**: Distinguish team-level from organization-level improvements.
**Strategy**: If same improvement appears in 3+ team retros, escalate to organization level. Local fixes for systemic problems waste effort.
**Application template**:
> Cross-reference improvement themes across teams for `{organization}`. Tag recurring themes as systemic. Recommend organizational intervention for systemic issues rather than team-by-team fixes. [PLAN]

## Meta-Strategy 3: Improvement Evidence Rigor
**Purpose**: Ensure improvements are genuinely effective, not placebo.
**Strategy**: Require before/after metrics for every improvement. Correlation is not causation — check for confounding factors.
**Application template**:
> For each improvement claiming positive impact, verify: Is the metric change statistically significant? Are there confounding factors? Would the metric have improved anyway? Only claim impact with [METRIC] evidence. [METRIC]
