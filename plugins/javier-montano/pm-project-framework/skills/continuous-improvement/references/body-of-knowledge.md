# Body of Knowledge — Continuous Improvement

## 1. Foundations
Continuous improvement (CI) is the systematic, ongoing effort to enhance products, processes, or services through incremental and breakthrough improvements. In PM, it ensures that project delivery capability improves over time. [DOC]

### 1.1 Key Frameworks
| Framework | Origin | Approach |
|-----------|--------|----------|
| Kaizen | Toyota Production System | Small, daily improvements [PLAN] |
| PDCA (Deming Cycle) | W. Edwards Deming | Plan-Do-Check-Act cycle [METRIC] |
| Lean Thinking | Womack/Jones | Eliminate waste, optimize flow [METRIC] |
| Retrospectives | Agile/Scrum | Sprint-based reflection and action [STAKEHOLDER] |
| Six Sigma DMAIC | Motorola/GE | Data-driven process improvement [METRIC] |

### 1.2 Improvement Categories
| Category | Focus | Typical Metrics |
|----------|-------|----------------|
| Process efficiency | Reduce waste, cycle time | Lead time, throughput [METRIC] |
| Quality | Reduce defects, rework | Defect rate, escape rate [METRIC] |
| Team performance | Velocity, satisfaction | Velocity trend, NPS [STAKEHOLDER] |
| Tool effectiveness | Adoption, automation | Tool usage, automation % [METRIC] |
| Communication | Stakeholder satisfaction | Response time, NPS [STAKEHOLDER] |

## 2. Improvement Identification Methods
1. **Retrospectives** — Sprint-based team reflection [STAKEHOLDER]
2. **Value stream mapping** — Visualize end-to-end process [METRIC]
3. **Root cause analysis** — 5-Why, fishbone diagrams [INFERENCIA]
4. **Metrics analysis** — Trend analysis on project KPIs [METRIC]
5. **Benchmarking** — Compare against industry or internal standards [DOC]

## 3. Improvement Prioritization
| Method | Criteria | Best For |
|--------|----------|---------|
| Impact/Effort Matrix | Quick visual prioritization | Small teams [PLAN] |
| A3 Problem Solving | Structured problem analysis | Complex problems [DOC] |
| RICE Scoring | Reach, Impact, Confidence, Effort | Backlog prioritization [METRIC] |
