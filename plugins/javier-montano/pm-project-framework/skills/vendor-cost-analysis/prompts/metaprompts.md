# Metaprompts — Vendor Cost Analysis

## Meta-Strategy 1: Hidden Cost Detection

**Purpose:** Ensure all cost categories are captured, especially hidden costs

```
For any vendor cost analysis, verify these hidden cost categories:
1. MANAGEMENT OVERHEAD — Hours spent managing vendor (meetings, reviews, escalations). Typically 10-20% of direct cost.
2. RAMP-UP COST — Weeks of reduced productivity during onboarding. Typically 4-8 weeks at 50% productivity.
3. QUALITY TAX — Rework, additional QA, defect resolution from vendor work. Track actual vs. internal defect rates.
4. KNOWLEDGE DRAIN — Cost of knowledge leaving when vendor engagement ends. Can the team maintain the code?
5. CONTEXT SWITCHING — Coordination costs when splitting work between internal and external teams.
If any category is missing, add it with an estimate. Flag estimates as [SUPUESTO] if no historical data.
```

## Meta-Strategy 2: Vendor Comparison Fairness Check

**Purpose:** Ensure vendor comparisons are apples-to-apples

```
When comparing vendors, verify:
1. Are scope definitions identical across all proposals?
2. Are pricing models normalized (T&M, fixed price, outcome-based all compared on same basis)?
3. Are quality assumptions equivalent (same testing standards, same Definition of Done)?
4. Are timeline assumptions aligned?
5. Are hidden costs estimated consistently across vendors?
Flag any comparison where normalization is incomplete. An unfair comparison leads to bad decisions.
```

## Meta-Strategy 3: Long-Term Cost Projection

**Purpose:** Prevent short-term cost optimization that creates long-term expense

```
Extend the cost analysis beyond the immediate engagement:
1. What are ongoing maintenance costs after vendor delivery?
2. Is knowledge transfer included in the contract?
3. What are switching costs if you need to change vendors later?
4. Does the vendor create dependencies (proprietary tools, frameworks)?
5. What is the cost trajectory if the engagement extends 50% beyond plan?
Present both short-term and 3-year TCO projections.
```
