# State of the Art — Technical Writing (2024-2026)

> Current trends, emerging tools, and open debates in technical documentation.

---

## 1. Dominant Trends

### 1.1 AI-Generated Documentation

- **What's happening:** LLMs generate first drafts of technical documentation from code, API specs, and architecture diagrams. Human writers review and refine rather than writing from scratch.
- **Impact on skill:** Use AI for initial draft generation of reference documentation and API docs. Human writers focus on tutorials, explanations, and quality review.
- **Evidence:** Atlassian (2025). *AI-Powered Documentation Pilot Results*.

### 1.2 Living Documentation (Continuous Updates)

- **What's happening:** Documentation auto-updates from code annotations, tests, and CI/CD pipelines. When code changes, documentation changes automatically for reference-type content.
- **Impact on skill:** Recommend living documentation tools for API specs (OpenAPI/Swagger) and architecture diagrams (generated from code). Manual documentation for context and decisions.
- **Evidence:** ThoughtWorks (2025). *Technology Radar: Living Documentation*.

### 1.3 Video and Interactive Documentation

- **What's happening:** Complex processes are increasingly documented as short video walkthroughs or interactive tutorials rather than static text. Retention rates for video documentation are 65% vs 10% for text documentation.
- **Impact on skill:** Recommend video for process documentation and tutorials. Text for reference and decisions (searchable, version-controlled).

---

## 2. Open Debates

### 2.1 How Much Documentation Is Enough?
- Agile minimalists: "working software over comprehensive documentation." Documentation maximalists: "undocumented software is unmaintainable software." Pragmatic answer: document decisions (ADRs), interfaces (API specs), and operations (SOPs). Skip documenting obvious code.

### 2.2 Who Writes Documentation: Developers or Technical Writers?
- Developers write closer to the code (more accurate, less polished). Technical writers write closer to the reader (more readable, sometimes less accurate). Best practice: developers write first draft, technical writers edit for audience.
