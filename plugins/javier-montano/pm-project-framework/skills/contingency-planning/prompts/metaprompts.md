# Metaprompts — Contingency Planning

## Meta-Strategy 1: Contingency Realism Check
**Purpose**: Ensure plans are executable, not theoretical.
**Strategy**: For each contingency, ask: Can we actually execute this in the time available? Do we have the resources? Has anyone tested it?
**Application template**:
> Review each contingency plan for `{project}`. Score executability (1-5) based on: resource availability, time to execute, team familiarity. Plans scoring <3 need revision or tabletop testing. [PLAN]

## Meta-Strategy 2: Cascade Impact Analysis
**Purpose**: Understand knock-on effects of contingency activation.
**Strategy**: When activating a contingency, map cascading impacts on other project dimensions.
**Application template**:
> For `{contingency_plan}` activation, map impacts on: budget (reserve consumption), schedule (timeline changes), scope (trade-offs), and team morale (change impact). Present full picture to decision-maker. [METRIC]

## Meta-Strategy 3: Reserve Adequacy Monitoring
**Purpose**: Track contingency reserve consumption and remaining coverage.
**Strategy**: Monitor reserve usage against remaining project risks. If reserves are consumed >50% before project midpoint, escalate.
**Application template**:
> Check reserve consumption for `{project}`. Calculate remaining reserve vs. remaining risk exposure. If reserve coverage ratio <1.0, recommend reserve replenishment or scope reduction. [METRIC]
