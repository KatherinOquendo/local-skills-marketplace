# Ceremony Facilitation — Metaprompts

## Metaprompt 1: Bias Check — Neutralidad del Facilitador

```
Revisa la guia de facilitacion generada y evalua:

1. **Neutralidad**: La guia dirige al grupo hacia un resultado predeterminado
   o permite que el grupo llegue a su propia conclusion?
2. **Inclusividad**: Las tecnicas seleccionadas favorecen un estilo de
   comunicacion (extrovertidos vs introvertidos, nativos vs no-nativos)?
3. **Formato bias**: Las tecnicas funcionan igual para participantes remotos
   y presenciales, o privilegian a un grupo?
4. **Cultural bias**: Las actividades asumen una cultura especifica
   (compartir emociones puede ser incomodo en culturas de alta distancia al poder)?

Para cada sesgo identificado:
- Describe el sesgo especifico
- Proporciona alternativa neutral
- Severidad: Alta (excluye participantes) / Media (desventaja) / Baja (preferencia)

Reescribe secciones con sesgo Alto o Medio.
```

## Metaprompt 2: Progressive Disclosure — Adaptacion por Experiencia

```
Adapta la guia de facilitacion al nivel de experiencia del facilitador:

**Nivel Novato (primera vez)**:
- Scripts literales de que decir en cada transicion
- Notas de timing con alarmas sugeridas
- "Si pasa X, haz Y" para los 5 escenarios mas comunes
- 1 tecnica de engagement (sin alternativas que confundan)

**Nivel Intermedio (5-20 ceremonias)**:
- Principios en lugar de scripts
- 2-3 tecnicas alternativas por seccion
- Anti-patrones avanzados
- Metricas de efectividad

**Nivel Avanzado (facilitador experimentado)**:
- Solo estructura y tiempos
- Innovacion y tecnicas nuevas
- Coaching points para desarrollar otros facilitadores
- Metricas de impacto organizacional

Genera la version para el nivel {LEVEL}.
```

## Metaprompt 3: Validation — Ceremonia Efectiva

```
Valida que la guia de facilitacion cumple criterios de efectividad:

[ ] Cada seccion tiene timebox explicito (no "segun sea necesario")
[ ] Check-in activity dura ≤10% del tiempo total
[ ] ≥2 tecnicas de engagement apropiadas para tamano del grupo y formato
[ ] Protocolo de conflicto con al menos 2 tecnicas de de-escalacion
[ ] Decision protocol declarado (consenso, consentimiento, mayoria, autoridad)
[ ] Closing incluye captura de action items Y feedback de ceremonia
[ ] Anti-patrones documentados con respuesta especifica
[ ] Guia usable por cualquier miembro del equipo, no solo Scrum Master
[ ] Materiales y herramientas listados con alternativas
[ ] Resultado esperado claramente articulado

Si algun criterio falla, reescribe la seccion.
Marca correcciones con [CORREGIDO] para trazabilidad.
```
