# Metaprompts — Risk Appetite Framework

## Meta-Strategy 1: Appetite-Behavior Alignment Check

**Purpose:** Verify stated appetite matches actual decision-making behavior

```
Compare stated risk appetite against actual project decisions:
1. Has the organization ever killed a project because risk exceeded appetite? If never → appetite may be performative.
2. Are projects routinely approved that exceed stated thresholds? If yes → appetite is too conservative on paper.
3. Do project teams know the appetite thresholds? If not → appetite is not operationalized.
Flag misalignments between stated and actual appetite.
```

## Meta-Strategy 2: Threshold Calibration

**Purpose:** Ensure thresholds are neither too tight (paralyzing) nor too loose (meaningless)

```
For each risk threshold:
1. How often has it been breached in the last 12 months? If never → may be too loose.
2. How often has it triggered unnecessary escalation? If frequently → may be too tight.
3. Is the threshold connected to an actual consequence? If not → it's just a number.
Calibrate thresholds to trigger action 10-20% of the time (meaningful but not overwhelming).
```

## Meta-Strategy 3: Stakeholder Appetite Divergence Detection

**Purpose:** Identify hidden conflicts in risk appetite across stakeholders

```
Survey key stakeholders on their individual risk appetite for each category:
1. Where do individual appetites diverge significantly from the stated organizational appetite?
2. Are sponsors more risk-hungry than the risk framework allows?
3. Are technical leads more risk-averse than the framework requires?
Document divergences and facilitate alignment conversations before they become project conflicts.
```
