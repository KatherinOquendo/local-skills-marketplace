# Body of Knowledge — Schedule Baseline

## 1. Definition and Scope

The schedule baseline is the approved version of the project schedule model used as the basis for comparison with actual results. It includes planned start/finish dates, milestones, dependencies, and resource assignments. Changes require formal change control. [DOC]

## 3. Schedule Development Process

1. **Define activities** — Decompose WBS work packages into schedulable activities [PLAN]
2. **Sequence activities** — Establish logical dependencies (FS, FF, SS, SF)
3. **Estimate durations** — Apply estimation techniques (analogous, parametric, 3-point)
4. **Develop schedule** — Apply CPM/CCM, assign resources, resolve conflicts
5. **Optimize schedule** — Resource leveling, fast-tracking, crashing
6. **Baseline approval** — Lock schedule as performance measurement baseline [SCHEDULE]

## 4. Key Schedule Metrics

| Metric | Formula | Target |
|--------|---------|--------|
| Schedule Performance Index (SPI) | EV / PV | 0.95-1.05 |
| Schedule Variance (SV) | EV - PV | ≥ 0 |
| Critical Path Length | Sum of CP activity durations | Minimize |
| Total Float | Latest finish - Earliest finish | > 0 for non-CP |
| Milestone adherence | On-time milestones / Total milestones | ≥ 85% [METRIC] |

## 5. Schedule Risk Integration

| Technique | Description | Output |
|-----------|-------------|--------|
| PERT analysis | Probabilistic duration estimates | Expected duration + variance |
| Monte Carlo | Simulation of schedule outcomes | P50/P80/P90 completion dates |
| Buffer sizing | Contingency and feeding buffers | Buffer allocation plan |
| Critical path analysis | Identify zero-float path | Focus area for risk management |
