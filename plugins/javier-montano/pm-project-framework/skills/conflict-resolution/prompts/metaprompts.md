# Metaprompts — Conflict Resolution

## Meta-Strategy 1: Neutrality Check
**Purpose**: Ensure conflict resolution approach is unbiased.
**Strategy**: Before facilitating, verify: Am I favoring one party? Am I projecting my preferences? Would both parties see this approach as fair?
**Application template**:
> Review `{resolution_plan}` for neutrality bias. Check if language favors either party. Ensure both perspectives are represented equally. If facilitator has a stake in the outcome, recommend external mediator. [STAKEHOLDER]

## Meta-Strategy 2: Root Cause vs. Symptom Distinction
**Purpose**: Address underlying issues, not surface disagreements.
**Strategy**: Use 5-Why technique on the conflict. Most visible conflicts are symptoms of deeper issues: unclear roles, resource constraints, or misaligned incentives.
**Application template**:
> For `{conflict}`, apply 5-Why analysis. If root cause is systemic (org design, process gap), recommend systemic fix alongside interpersonal resolution. Resolving only the symptom guarantees recurrence. [INFERENCIA]

## Meta-Strategy 3: Relationship Preservation Priority
**Purpose**: Ensure resolution preserves working relationships for project duration.
**Strategy**: Test resolution against: Will both parties be able to collaborate effectively after this? If resolution creates winners and losers, the project loses.
**Application template**:
> Evaluate `{proposed_resolution}` for relationship impact. Score future collaboration likelihood (1-5) for each party. If either party scores <3, revise approach toward collaborative or compromising mode. [STAKEHOLDER]
