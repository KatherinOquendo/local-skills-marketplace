# State of the Art — Definition of Done (2024-2026)

## Tendencia 1: Automated DoD Verification
Pipelines CI/CD que verifican automáticamente criterios de DoD: cobertura de tests, code review status, security scans, documentation checks. El DoD se convierte en un quality gate del pipeline, no solo un checklist manual. [PLAN]

## Tendencia 2: AI-Assisted Acceptance Criteria Generation
Herramientas AI que generan acceptance criteria a partir de user stories, sugiriendo criterios que el equipo puede haber omitido. Reducen el gap entre DoR ideal y DoR real al detectar ambigüedades. [INFERENCIA]

## Tendencia 3: Progressive DoD Maturity Models
Frameworks que definen niveles de madurez del DoD (Level 1: básico, Level 2: automatizado, Level 3: predictivo). Equipos avanzan gradualmente en lugar de intentar un DoD perfecto desde el día 1. [PLAN]

## Tendencia 4: Cross-Team DoD Harmonization
En organizaciones con múltiples equipos, DoD mínimo organizacional + DoD extendido por equipo. Herramientas de governance verifican que todos los equipos cumplan el baseline sin imponer uniformidad excesiva. [STAKEHOLDER]

## Tendencia 5: DoD como Contrato de Servicio Interno
DoD tratado como SLA interno entre desarrollo y operaciones/negocio. Métricas de compliance de DoD publicadas como indicadores de calidad de servicio del equipo. [METRIC]

## Implicaciones para PMO-APEX

| Tendencia | Impacto en APEX | Acción Recomendada |
|-----------|-----------------|-------------------|
| Automated verification | DoD verificable por pipeline | Integrar checks en CI/CD |
| AI-assisted criteria | Mejor DoR quality | Usar prompts para generar criteria |
| Progressive maturity | DoD escalable | Definir niveles L1/L2/L3 |
| Cross-team harmonization | Consistencia portfolio | Establecer DoD baseline organizacional |
| DoD como SLA | Accountability medible | Publicar compliance metrics |
