# Metaprompts — Program Management

## Meta-Strategy 1: Interdependency Bias Check

**Purpose:** Ensure program-level analysis doesn't overlook hidden dependencies

```
Before finalizing any program plan or assessment, apply this bias check:
1. HIDDEN DEPENDENCIES — What cross-project dependencies are not explicitly documented?
2. OPTIMISM BIAS — Are benefit estimates based on best-case scenarios? Apply -15% adjustment.
3. RESOURCE ILLUSION — Are shared resources counted at >80% availability? Flag as [SUPUESTO].
4. STAKEHOLDER BLIND SPOTS — Which stakeholder groups have NOT been consulted?
5. INTEGRATION RISK — What happens if Component X is 3 sprints late? Model the cascade.
Re-evaluate the analysis after applying each check. Document changes made.
```

## Meta-Strategy 2: Progressive Disclosure for Program Reporting

**Purpose:** Layer program information for different audience levels

```
Structure program outputs in 3 layers:
LAYER 1 (Executive — 1 page): RAG status, key decisions needed, benefit trajectory
LAYER 2 (Program Board — 3-5 pages): Component analysis, risk heat map, resource status
LAYER 3 (Program Team — detailed): Full dependency maps, detailed metrics, action items

For each layer:
- Verify that higher layers are consistent with lower layers
- Ensure no detail in Layer 1 contradicts Layer 3
- Tag information source at each layer
Apply this structure to: {current_deliverable}
```

## Meta-Strategy 3: Adversarial Program Scenario Testing

**Purpose:** Stress-test program assumptions through adversarial thinking

```
For the current program plan, generate 3 adversarial scenarios:
SCENARIO A (Resource Crisis): 30% of shared resources become unavailable in Sprint 5
SCENARIO B (Scope Explosion): Regulatory change adds 2 new component projects mid-program
SCENARIO C (Benefit Collapse): Primary benefit driver loses 50% of expected value

For each scenario:
1. Impact assessment on program timeline and benefits
2. Trigger indicators (early warning signals)
3. Response strategy with decision points
4. Minimum viable program (what can still be delivered?)
Tag all assumptions explicitly.
```
