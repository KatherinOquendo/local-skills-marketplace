# Body of Knowledge — Program Management

## 1. Definition and Scope

Program management is the coordinated management of multiple related projects and activities to obtain benefits not available from managing them individually. Unlike project management (single deliverable focus), program management orchestrates interdependencies across a portfolio of initiatives. [DOC]

## 3. Program Lifecycle Phases

### 3.1 Program Definition
- Vision and mission alignment with strategic objectives
- Program charter development and stakeholder mapping
- Benefit identification and initial business case [PLAN]

### 3.2 Program Planning
- Roadmap construction with milestone dependencies
- Resource allocation across constituent projects
- Risk aggregation and cross-project impact analysis [PLAN]

### 3.3 Program Delivery
- Component project initiation and oversight
- Benefit realization tracking and value delivery
- Change management across the program ecosystem [METRIC]

### 3.4 Program Closure
- Benefit sustainment planning
- Knowledge transfer and lessons consolidation
- Organizational capability transition [PLAN]

## 4. Governance Structures

### 4.1 Program Board
The program board provides strategic direction and resolves escalated issues. Membership typically includes:
- Program sponsor (executive accountability)
- Program manager (operational leadership)
- Senior business representatives
- Key technical authorities [STAKEHOLDER]

### 4.2 Decision Rights Matrix

| Decision Type | Authority Level | Escalation Path |
|--------------|----------------|-----------------|
| Scope change (<5% budget impact) | Program Manager | Program Board |
| Scope change (>5% budget impact) | Program Board | Portfolio Board |
| Resource reallocation (within program) | Program Manager | None |
| Resource reallocation (cross-program) | Portfolio Board | Executive Committee |
| Benefit target adjustment | Program Board | Executive Committee |

## 5. Benefit Management

Benefits are the measurable improvements resulting from program outcomes. The benefit management cycle includes:

1. **Identification** — Map benefits to strategic objectives
2. **Analysis** — Quantify expected value and timeline
3. **Planning** — Define realization approach and owners
4. **Realization** — Track actual vs. planned benefits
5. **Transition** — Hand over to operational benefit owners [METRIC]

## 6. Stakeholder Engagement in Programs

Program-level stakeholder engagement differs from project-level due to:
- Multiple stakeholder communities across projects
- Political dynamics between project teams
- Strategic stakeholder influence on benefit realization
- External stakeholder management (regulators, partners) [STAKEHOLDER]

## 7. Risk Management at Program Level

| Risk Category | Example | Mitigation Approach |
|--------------|---------|-------------------|
| Interdependency risk | Project A delay impacts Project B | Buffer management, critical chain |
| Benefit erosion | Market shift reduces expected value | Benefit review gates |
| Resource contention | Shared specialists overallocated | Resource pooling, capacity planning |
| Integration risk | Components don't integrate | Architecture governance |
| Organizational change | Resistance to transformation | Change management program |

## 8. Program Metrics and KPIs

| Metric | Formula | Target |
|--------|---------|--------|
| Benefit Realization Index | Actual benefits / Planned benefits | ≥ 0.90 |
| Program Health Score | Weighted average of component project health | ≥ 7.5/10 |
| Schedule Performance (Program) | Milestones achieved on time / Total milestones | ≥ 85% |
| Resource Utilization | Allocated hours / Available hours | 75-85% |
| Stakeholder Satisfaction | Survey score | ≥ 4.0/5.0 |

## 9. Tools and Techniques

- Program roadmap with dependency visualization
- Benefits dependency network (BDN)
- Program RAID log (Risks, Assumptions, Issues, Dependencies)
- Integrated milestone plan
- Program dashboard and reporting cadence [PLAN]
