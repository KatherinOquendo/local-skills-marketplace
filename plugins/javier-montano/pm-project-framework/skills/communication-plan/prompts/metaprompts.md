# Metaprompts — Communication Plan

## Meta-Strategy 1: Audience-First Message Design

**Purpose**: Ensure every communication answers "So what?" for its audience.

**Strategy**:
Before sending any communication, test it against the audience filter:
1. What decision does this stakeholder need to make?
2. What action should they take after reading this?
3. What is the single most important message?

**Application template**:
> Review `{communication}` for `{audience}`. Strip all information that does not support a decision or action. Lead with the key message. If the communication exceeds 1 page, create a TL;DR. [STAKEHOLDER]

## Meta-Strategy 2: Communication Debt Tracking

**Purpose**: Prevent accumulation of overdue or missing communications.

**Strategy**:
Track communication commitments like project tasks:
- Overdue updates = communication debt
- Skipped meetings = engagement debt
- Unanswered questions = trust debt

**Application template**:
> Audit communication plan execution for `{project}`. List all overdue communications, skipped meetings, and unanswered stakeholder questions. Prioritize debt repayment by stakeholder influence. Never let communication debt exceed 1 sprint. [PLAN]

## Meta-Strategy 3: Feedback Loop Verification

**Purpose**: Ensure communications are received and understood.

**Strategy**:
For each critical communication, verify:
- Was it received? (open/read confirmation)
- Was it understood? (follow-up questions or actions)
- Did it produce the intended outcome? (decision made, action taken)

**Application template**:
> For `{critical_communication}`, implement 3-step verification. If no response within 48h, follow up through alternative channel. Document communication effectiveness for plan optimization. [METRIC]
