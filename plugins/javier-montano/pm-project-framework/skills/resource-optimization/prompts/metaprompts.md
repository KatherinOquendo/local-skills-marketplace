# Metaprompts — Resource Optimization

## Meta-Strategy 1: Human-Centric Optimization Check

**Purpose:** Ensure optimization doesn't sacrifice team wellbeing for efficiency

```
For any resource optimization output, verify:
1. No individual exceeds 85% utilization for more than 2 consecutive sprints
2. Context switching is limited to ≤ 2 projects per person
3. Learning/growth time (10-15%) is preserved in allocation
4. Vacation and buffer time is not consumed by optimization
5. Single-point-of-failure risks are addressed (no solo experts on critical path)
Flag violations as [SUPUESTO] — optimization at cost of sustainability.
```

## Meta-Strategy 2: Optimization Sensitivity Testing

**Purpose:** Test robustness of resource plan against disruptions

```
Stress-test the optimized plan:
1. What happens if the most-allocated person is unavailable for 2 weeks?
2. What happens if a new high-priority project consumes 20% of total capacity?
3. What happens if 2 team members resign simultaneously?
For each scenario, identify impact and pre-position contingency responses.
```

## Meta-Strategy 3: Utilization vs. Throughput Balance

**Purpose:** Prevent the trap of high utilization with low throughput

```
Verify optimization prioritizes throughput over utilization:
1. Is WIP (work in progress) limited? High utilization + high WIP = low throughput.
2. Are buffers preserved? 100% utilization = 0 flexibility = delays cascade.
3. Is flow efficiency measured? Value-add time / total time should be > 40%.
If utilization > 85% but flow efficiency < 30%, the optimization is counter-productive.
```
