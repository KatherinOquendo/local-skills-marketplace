# Body of Knowledge — RACI Matrix

## 1. Definition and Scope

A RACI matrix (Responsible, Accountable, Consulted, Informed) is a responsibility assignment chart that maps project activities or deliverables to stakeholder roles, clarifying decision rights and eliminating ambiguity about who does what. [DOC]

## 2. Foundational Frameworks

| Framework | Source | Key Contribution |
|-----------|--------|-----------------|
| PMBOK Resource Management | PMI, 2021 | Responsibility assignment matrices |
| COBIT 2019 | ISACA | Governance RACI charts |
| ITIL 4 | AXELOS | Service management responsibility mapping |
| PRINCE2 | AXELOS | Role-based project governance |
| Organizational Design | Galbraith, 2014 | Decision rights frameworks |

## 3. RACI Roles Defined

| Role | Definition | Rules |
|------|-----------|-------|
| **R** — Responsible | Does the work to complete the task | Multiple R per task allowed |
| **A** — Accountable | Ultimately answerable; approves/signs off | Exactly ONE A per task |
| **C** — Consulted | Provides input before work is done (two-way) | As needed |
| **I** — Informed | Notified after work is done (one-way) | As needed |

## 4. RACI Variants

| Variant | Additions | Use Case |
|---------|-----------|----------|
| RASCI | S = Supportive | When support roles need distinction |
| DACI | D = Driver | Decision-focused frameworks |
| CAIRO | O = Omitted (explicitly excluded) | When exclusion matters |
| RAPID | Recommend, Agree, Perform, Input, Decide | Complex decision environments |

## 5. Construction Process

1. **List activities/deliverables** — Rows of the matrix
2. **Identify roles/people** — Columns of the matrix
3. **Assign RACI codes** — One code per cell
4. **Validate rules** — One A per row, at least one R per row
5. **Review with stakeholders** — Confirm understanding and agreement
6. **Publish and maintain** — Living document, updated with scope changes [PLAN]

## 6. Validation Rules

| Rule | Check | Remedy |
|------|-------|--------|
| No A assigned | Task has no accountability | Assign exactly one A |
| Multiple A's | Shared accountability = no accountability | Consolidate to one A |
| No R assigned | Nobody doing the work | Assign at least one R |
| Too many C's | Decision bottleneck (consultation overload) | Reduce to essential consultees |
| A without R | Accountable person not working or overseeing | Verify A has visibility |
| All I's in a column | Role may be unnecessary | Consider removing from matrix |

## 7. Common Anti-Patterns

| Anti-Pattern | Problem | Solution |
|-------------|---------|----------|
| RACI by committee | Everyone is C or R | Apply minimum necessary principle |
| Accountability void | No A assigned | Enforce one-A rule |
| Phantom roles | Names on matrix but not engaged | Verify participation |
| Static RACI | Never updated after creation | Review at each phase gate |
| RACI as politics | Roles assigned for optics, not function | Align to actual work |

## 8. Integration with Other PM Tools

- **WBS** — RACI rows align to WBS work packages
- **Stakeholder Register** — RACI columns derived from stakeholder list
- **Communication Plan** — I and C assignments inform communication needs
- **Quality Plan** — Review and approval roles mapped to A assignments [PLAN]
