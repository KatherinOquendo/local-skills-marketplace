# Metaprompts — RACI Matrix

## Meta-Strategy 1: Accountability Clarity Test

**Purpose:** Ensure accountability assignments are real, not ceremonial

```
For each A (Accountable) assignment, verify:
1. Does this person actually have AUTHORITY to make decisions for this activity?
2. Does this person have VISIBILITY into the activity's progress?
3. Can this person BLOCK progress if quality is insufficient?
4. Is this the SAME person the team would naturally escalate to?
If any answer is No, the A assignment is likely ceremonial. Reassign to the actual decision-maker.
```

## Meta-Strategy 2: Consultation Overload Detection

**Purpose:** Prevent decision paralysis from excessive consultation

```
Analyze the C (Consulted) column for each activity:
1. Count total C assignments per activity. If > 3, flag as consultation overload.
2. For each C, ask: "Would the quality of the decision meaningfully degrade without this person's input?"
3. If the answer is "marginal impact" → downgrade C to I (Informed).
4. Check: Is the C being consulted because of politics or actual expertise?
Target: Average ≤ 2 C's per activity.
```

## Meta-Strategy 3: Role Balance Visualization

**Purpose:** Identify overloaded and underutilized roles

```
For the complete RACI matrix:
1. Count R+A assignments per role. If > 40% of activities, flag as overloaded.
2. Count roles with only I assignments. Question if they need to be on the matrix.
3. Identify roles that are A but never R. Are they empowered or disconnected?
4. Check for "RACI shadow" — unofficial responsibilities not captured in the matrix.
Generate a role workload heat map and rebalancing recommendations.
```
