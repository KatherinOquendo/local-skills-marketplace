# Metaprompts — Quality Plan

## Meta-Strategy 1: Cost of Quality Balance Check

**Purpose:** Ensure quality plan balances prevention investment against failure costs

```
For the quality plan, validate cost balance:
1. PREVENTION vs. DETECTION — Is the plan investing more in preventing defects than finding them?
2. COVERAGE vs. COST — Are quality activities proportional to deliverable risk/value?
3. AUTOMATION vs. MANUAL — Can any manual quality activity be automated for better ROI?
4. OVERHEAD LIMIT — Total quality effort should be 15-25% of project effort. Verify.
Adjust the plan if any dimension is unbalanced.
```

## Meta-Strategy 2: Stakeholder Quality Expectation Alignment

**Purpose:** Verify quality plan addresses all stakeholder quality perspectives

```
Validate quality plan against stakeholder expectations:
CUSTOMER: Does the plan ensure deliverables meet functional and non-functional requirements?
SPONSOR: Does the plan provide confidence in quality without excessive cost?
TEAM: Are quality activities achievable within capacity without burnout?
REGULATOR: Are all compliance requirements explicitly addressed?
For any misalignment, adjust the quality plan and document trade-offs.
```

## Meta-Strategy 3: Quality Plan Testability Check

**Purpose:** Ensure every quality objective is measurable and verifiable

```
For each quality objective in the plan:
1. Is there a METRIC defined? (If not, add one)
2. Is the metric AUTOMATED or manually collected? (Prefer automated)
3. Is the THRESHOLD clearly defined with green/amber/red zones?
4. Is the OWNER assigned for metric monitoring?
5. Is the RESPONSE defined when threshold is breached?
Remove or rewrite any quality objective that cannot be measured.
```
