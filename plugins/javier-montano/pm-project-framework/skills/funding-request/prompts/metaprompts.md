# Metaprompts — Funding Request

## Metaprompt 1: Optimism Bias Detector

**Strategy**: Apply reference-class forecasting to challenge cost estimates in funding requests.

**Protocol**:
1. Identify the project type and size category
2. Retrieve historical cost overrun percentages for similar project classes
3. Apply Flyvbjerg's reference-class adjustment factors
4. Generate a bias-adjusted funding range with confidence intervals
5. Present both the original and adjusted figures with [INFERENCIA] tags

**Activation**:
> "Before finalizing this funding request, apply reference-class forecasting. Compare our estimates against historical data for similar {PROJECT_TYPE} projects. What is the probability-adjusted budget range at 50%, 80%, and 95% confidence levels?"

## Metaprompt 2: Adversarial Stakeholder Simulation

**Strategy**: Pre-test the funding request against likely objections from different stakeholder archetypes.

**Protocol**:
1. Identify the approval body composition
2. Simulate each stakeholder archetype (CFO skeptic, tech optimist, risk-averse board member)
3. Generate top 3 objections per archetype
4. Draft counter-arguments with supporting evidence
5. Strengthen the proposal based on simulated pushback

**Activation**:
> "Simulate a funding review session for this proposal. Play the roles of: (a) a skeptical CFO focused on cash flow, (b) a board member concerned about strategic fit, (c) a PMO director worried about resource capacity. Generate their likely objections and strengthen the proposal accordingly."

## Metaprompt 3: Progressive Disclosure Funding Narrative

**Strategy**: Structure the funding request in layers of detail for different audience attention spans.

**Protocol**:
1. Layer 0 — One-line ask (elevator pitch)
2. Layer 1 — Executive summary (1 paragraph)
3. Layer 2 — Key financials table (half page)
4. Layer 3 — Full business case with appendices
5. Each layer must stand alone as a complete argument

**Activation**:
> "Structure this funding request using progressive disclosure. Create four layers of detail, each self-contained. A busy executive should be able to approve at Layer 1; a diligent reviewer should find full evidence at Layer 3."
