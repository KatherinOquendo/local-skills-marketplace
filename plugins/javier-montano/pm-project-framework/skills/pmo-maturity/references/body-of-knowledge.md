## 1. Fundamentos Teóricos

### 1.1 Modelos de Madurez
Los modelos de madurez proporcionan un framework para evaluar la sofisticación de las prácticas de PM en una organización. Los dos principales son [DOC]:

**OPM3 (Organizational Project Management Maturity Model)**
- Desarrollado por PMI
- Evalúa madurez en 3 dominios: Project, Program, Portfolio
- 4 niveles de mejora: Standardize, Measure, Control, Continuously Improve
- Incluye ~600 best practices organizadas jerárquicamente

**P3M3 (Portfolio, Programme, Project Management Maturity Model)**
- Desarrollado por Axelos (PRINCE2/PRINCE2 Agile)
- 5 niveles de madurez (1-5) por perspectiva
- 7 perspectivas: Management Control, Benefits Management, Financial Management, Stakeholder Management, Risk Management, Organizational Governance, Resource Management

### 1.2 Escala de Madurez (5 Niveles)
| Nivel | Nombre | Descripción |
|-------|--------|-------------|
| 1 | Awareness | Procesos ad-hoc, éxito depende de individuos |
| 2 | Repeatable | Procesos básicos establecidos, resultados repetibles |
| 3 | Defined | Procesos estandarizados, documentados, y seguidos |
| 4 | Managed | Procesos medidos y controlados con métricas |
| 5 | Optimized | Mejora continua basada en datos y feedback |

### 1.3 Dimensiones de Evaluación
Las dimensiones comunes entre OPM3 y P3M3:
1. **Procesos** — Estandarización y documentación de procesos PM
2. **Personas** — Competencias, roles, y capacitación
3. **Herramientas** — Tecnología de soporte a PM
4. **Governance** — Estructuras de decisión y oversight
5. **Cultura** — Valores y comportamientos organizacionales respecto a PM

## 2. Metodología de Evaluación

### 2.1 Instrumentos de Recolección
| Instrumento | Datos que Produce | Tamaño Mínimo |
|-------------|-------------------|---------------|
| Entrevistas estructuradas | Datos cualitativos profundos | 8-12 entrevistas |
| Encuesta cuantitativa | Datos escalares por dimensión | 30+ respondentes |
| Revisión documental | Evidencia de procesos y estándares | 10+ documentos |
| Observación directa | Validación de prácticas declaradas | 3-5 sesiones |

### 2.2 Calibración de Scores
Cada score debe tener evidencia específica:
- **Nivel 1**: No existe proceso documentado → evidencia de ausencia
- **Nivel 2**: Proceso existe pero es inconsistente → evidencia de variabilidad
- **Nivel 3**: Proceso documentado y seguido → evidencia de documentación + compliance
- **Nivel 4**: Proceso medido con KPIs → evidencia de métricas y dashboards
- **Nivel 5**: Proceso optimizado con feedback loop → evidencia de mejoras implementadas

## 3. Gap Analysis

### 3.1 Current vs. Target
Para cada dimensión: Score_actual vs. Score_objetivo = Gap
Priorizar gaps por: impacto estratégico (alto/medio/bajo) x factibilidad (alta/media/baja)

### 3.2 Effort Estimation
Progresión típica de un nivel requiere:
- Nivel 1→2: 6-12 meses, 4-8 FTE-meses [HISTORICO]
- Nivel 2→3: 12-18 meses, 8-16 FTE-meses [HISTORICO]
- Nivel 3→4: 18-24 meses, 12-24 FTE-meses [HISTORICO]
- Nivel 4→5: 24-36 meses, 16-32 FTE-meses [HISTORICO]

## 4. Roadmap Design

### 4.1 Estructura Típica
- **Quick wins (0-3 meses)**: Templates, training, basic tools
- **Foundation (3-12 meses)**: Process standardization, governance establishment
- **Measurement (12-24 meses)**: KPIs, dashboards, data-driven decisions
- **Optimization (24+ meses)**: Continuous improvement, innovation

### 4.2 Secuenciación
Las dimensiones tienen dependencias: no se puede medir (Nivel 4) lo que no está definido (Nivel 3). El roadmap respeta estas dependencias naturales [PLAN].

## 5. Anti-patrones

| Anti-patrón | Consecuencia | Corrección |
|-------------|-------------|------------|
| Apuntar a Nivel 5 sin Nivel 3 | Frustración y desperdicio | Progresión incremental |
| Invertir uniformemente | ROI bajo en dimensiones no críticas | Priorización estratégica |
| Maturity assessment sin follow-up | Ejercicio académico | Commitment ejecutivo antes de evaluar |
| Copiar roadmap de otra organización | Contexto diferente, resultados diferentes | Roadmap basado en gaps propios |
| Evaluar sin benchmark | Scores sin contexto | Incluir benchmark sectorial |
