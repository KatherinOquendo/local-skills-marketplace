# Use-Case Prompts — Scrum Framework

## Prompt 1: Scrum Implementation Guide

**When:** Setting up Scrum for a new team or project
**Context variables:** `{team_name}`, `{team_size}`, `{product_vision}`, `{sprint_length}`
**Deliver:**

```
Design a Scrum implementation for {team_name}: Size: {team_size}, Vision: {product_vision}, Sprint length: {sprint_length}. Include: role assignments, ceremony calendar with timeboxes, artifact templates, Definition of Done, Definition of Ready, working agreements, and success metrics for first 3 sprints.
```

## Prompt 2: Sprint Planning Facilitation

**When:** Preparing or facilitating sprint planning
**Context variables:** `{sprint_number}`, `{team_velocity}`, `{backlog_items}`, `{capacity}`
**Deliver:**

```
Facilitate sprint planning for Sprint {sprint_number}: Velocity: {team_velocity}, Candidate items: {backlog_items}, Capacity: {capacity}. Produce: sprint goal, selected backlog items with task breakdown, capacity allocation, identified risks, and sprint forecast.
```

## Prompt 3: Scrum Health Assessment

**When:** Evaluating how well Scrum is being practiced
**Context variables:** `{team_name}`, `{sprint_metrics}`, `{ceremony_attendance}`, `{stakeholder_feedback}`
**Deliver:**

```
Assess Scrum health for {team_name}: Metrics: {sprint_metrics}, Ceremony data: {ceremony_attendance}, Stakeholder feedback: {stakeholder_feedback}. Evaluate adherence to Scrum Guide, identify anti-patterns, and recommend improvements with expected impact.
```
