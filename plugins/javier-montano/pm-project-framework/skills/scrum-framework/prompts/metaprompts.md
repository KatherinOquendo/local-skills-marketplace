# Metaprompts — Scrum Framework

## Meta-Strategy 1: Scrum Anti-Pattern Detection

**Purpose:** Identify common Scrum implementation failures

```
Check for these anti-patterns:
1. ZOMBIE SCRUM — All ceremonies performed but no real adaptation happening
2. SPRINT SCOPE CHANGE — Work added/removed mid-sprint without sprint goal change
3. ABSENT PRODUCT OWNER — PO not available during sprint, decisions delayed
4. ESTIMATION THEATER — Story points used for performance measurement, not planning
5. RETRO WITHOUT ACTION — Retrospective produces insights but no implemented changes
For each detected pattern, recommend specific corrective action.
```

## Meta-Strategy 2: Outcome vs. Output Validation

**Purpose:** Ensure Scrum is delivering value, not just velocity

```
For the last 3 sprints, verify:
1. What BUSINESS OUTCOME did each sprint deliver? (Not just stories completed)
2. Did stakeholders USE the increment? If not, why?
3. Are sprint goals connected to measurable product/business metrics?
4. Is velocity trending but customer satisfaction flat or declining?
If output is high but outcomes are low, refocus on sprint goal quality.
```

## Meta-Strategy 3: Scrum Master Effectiveness

**Purpose:** Evaluate whether Scrum Master role is delivering value

```
Assess SM effectiveness:
1. Are impediments being resolved within 24-48 hours?
2. Is the team's self-organization improving sprint over sprint?
3. Are retrospective actions being implemented?
4. Is the SM coaching the team or just facilitating ceremonies?
5. Is the SM protecting the team from external interference?
Rate each area and identify SM development priorities.
```
