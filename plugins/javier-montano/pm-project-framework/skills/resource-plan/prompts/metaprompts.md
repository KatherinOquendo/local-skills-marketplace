# Metaprompts — Resource Plan

## Meta-Strategy 1: Single-Point-of-Failure Detection

**Purpose:** Identify critical knowledge dependencies that threaten project continuity

```
For each critical skill in the resource plan:
1. How many people possess this skill? If = 1, flag as SPOF.
2. Is there a documented backup person? If No, create cross-training plan.
3. What is the impact if this person is unavailable for 2 weeks?
4. Is critical knowledge documented or only in someone's head?
Target: No critical skill depends on a single person.
```

## Meta-Strategy 2: Sustainable Pace Validation

**Purpose:** Prevent burnout-inducing resource plans

```
Validate sustainable pace:
1. No resource allocated > 85% for > 3 consecutive sprints
2. Vacation time preserved (not consumed by project demands)
3. Learning/development time (10%) is explicitly allocated
4. Buffer for unplanned work (15%) is included
If any violation exists, rebalance the plan before approval.
```

## Meta-Strategy 3: Acquisition Lead Time Reality Check

**Purpose:** Ensure resource availability assumptions are realistic

```
For each external resource in the plan:
1. What is the realistic lead time to acquire? (hiring: 8-12 weeks, contractor: 2-4 weeks)
2. What is the onboarding time to productivity? (internal: 1-2 weeks, external: 3-4 weeks)
3. Is the acquisition start date early enough to meet the need date?
If lead time + onboarding > time available, flag as [SUPUESTO] and propose alternatives.
```
