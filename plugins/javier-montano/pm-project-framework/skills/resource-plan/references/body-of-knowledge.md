# Body of Knowledge — Resource Plan

## 1. Definition and Scope

A resource plan identifies, documents, and schedules all resources (human, material, equipment, financial) required to execute a project. It translates the WBS and schedule into specific resource requirements with timelines, roles, skills, and allocation percentages. [DOC]

## 3. Resource Plan Components

### 3.1 Resource Requirements Matrix
- Role definitions and competency requirements
- Headcount per phase/sprint
- Allocation percentages per resource
- Skill requirements per work package [PLAN]

### 3.2 Resource Calendar
- Availability windows per resource
- Vacation, training, and non-project commitments
- Ramp-up and ramp-down schedules
- Organizational holidays and constraints [SCHEDULE]

### 3.3 Acquisition Strategy
- Internal vs. external sourcing decisions
- Staff augmentation criteria and triggers
- Onboarding timeline and knowledge transfer plan
- Procurement lead times for equipment/tools [PLAN]

## 4. Resource Estimation Techniques

| Technique | Description | Accuracy | Effort |
|-----------|-------------|----------|--------|
| Analogous | Based on similar past projects | ±25% | Low |
| Parametric | Mathematical models (effort per story point) | ±15% | Medium |
| Bottom-up | Activity-level estimation rolled up | ±10% | High |
| Three-point | Optimistic/most likely/pessimistic | ±10-15% | Medium |
| Expert judgment | SME consultation | Variable | Low |

## 5. Team Structure Patterns

| Pattern | Description | Best For |
|---------|-------------|----------|
| Functional | Resources from functional departments | Waterfall, matrix organizations |
| Cross-functional | Dedicated team with all needed skills | Agile, product teams |
| Hybrid | Core team + functional specialists | Large programs |
| Distributed | Geographically dispersed team | Global projects |
| Scaled | Multiple coordinated teams | SAFe, LeSS, programs |

## 6. Key Metrics

| Metric | Formula | Target |
|--------|---------|--------|
| Resource availability | Available hours / Required hours | ≥ 100% |
| Skill coverage | Required skills covered / Total required | 100% |
| Onboarding velocity | Days to full productivity | ≤ 15 days |
| Resource plan accuracy | Actual vs. planned resource usage | ±15% |
| Team stability index | Members staying / Total members per quarter | ≥ 85% |
