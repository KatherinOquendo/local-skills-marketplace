# State of the Art — Resource Plan (2024-2026)

## Trend 1: Skills-First Resource Planning

Shifting from role-based planning ("need 5 developers") to skills-based planning ("need Python + AWS + event-driven architecture expertise"). AI tools match skill profiles to project needs. [DOC]

> "Principio Rector: Planificar recursos es planificar el éxito del equipo."

## Trend 2: Dynamic Staffing Models

Fixed-team models replaced by fluid staffing where resources are allocated based on real-time demand signals. Gig-economy principles applied to enterprise project staffing. [INFERENCIA]

## Trend 3: AI-Assisted Workforce Planning

Predictive models forecast resource needs 2-4 quarters ahead based on project pipeline, skill trends, and attrition patterns. Organizations proactively build capability before demand. [DOC]

## Trend 4: Wellbeing-Integrated Resource Plans

Resource plans now include capacity buffers for learning, wellness, and innovation. Sustainable pace (80% allocation maximum) is becoming a planning constraint, not an aspiration. [INFERENCIA]

## Trend 5: Platform Team Resource Models

Team Topologies influence resource planning with dedicated platform teams, enabling teams, and stream-aligned teams each having distinct resource planning patterns. [DOC]

| Team Type | Planning Pattern | Allocation Model |
|-----------|-----------------|-----------------|
| Stream-aligned | Stable, long-lived | 100% dedicated |
| Platform | Shared services | Request-based |
| Enabling | Temporary engagement | Time-boxed |
| Complicated subsystem | Specialist pool | As-needed |
