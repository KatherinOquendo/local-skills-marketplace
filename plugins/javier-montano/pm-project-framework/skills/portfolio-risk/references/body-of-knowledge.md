# Portfolio Risk Management — Body of Knowledge

## Standards & Frameworks
Core knowledge areas: Portfolio risk aggregation, inter-project dependencies, systemic risk, concentration risk, risk appetite

## PMBOK Alignment
- Integrates with PMBOK 7th Edition performance domains
- Supports ISO 21502:2020 project governance requirements
- Compatible with PRINCE2 and hybrid methodologies

## Key Concepts

### Inputs
- Project charter and scope statement
- Stakeholder register and engagement plan
- Risk register and assumption log
- Historical data from similar projects

### Process Steps
1. Assess current state and readiness
2. Define target state with measurable criteria
3. Gap analysis between current and target
4. Design intervention strategy
5. Implement with monitoring and feedback loops
6. Measure outcomes against success criteria
7. Adjust and iterate based on results

### Outputs
- Assessment report with evidence-tagged findings
- Recommendations with priority ranking
- Implementation roadmap with milestones
- Success metrics and measurement plan

## Quality Criteria
| Criterion | Threshold | Evidence Tag |
|-----------|-----------|-------------|
| Evidence coverage | ≥70% [PLAN]/[METRIC] | Required |
| Assumptions | <30% [SUPUESTO] | Warning if exceeded |
| Stakeholder validation | ≥1 [STAKEHOLDER] source | Required |
| Historical basis | ≥1 [HISTORICO] reference | Recommended |

## Integration Points
- Feeds into: risk register, project charter, steering committee review
- Receives from: stakeholder register, assumption log, historical data
- Cross-references: methodology playbook, quality plan, communication plan
