# Body of Knowledge — Benefits Realization Plan

## 1. Foundations

Benefits realization management (BRM) ensures that project outcomes deliver measurable value to the organization. It bridges the gap between project delivery and strategic objectives. [DOC]

### 1.1 Key Concepts

| Concept | Definition | Source |
|---------|-----------|--------|
| Benefit | A measurable improvement from project outcomes | PMI Standard for BRM |
| Dis-benefit | A negative outcome that must be managed | PRINCE2 |
| Benefit Owner | Stakeholder accountable for realizing a specific benefit | PMI |
| Benefit Profile | Detailed description of a single benefit | MSP Framework |
| Realization Timeline | When benefits are expected to materialize | ISO 21505 |

### 1.2 Benefit Categories

| Category | Examples | Measurement Approach |
|----------|---------|---------------------|
| Financial | Cost reduction, revenue increase | FTE-months saved, throughput increase [METRIC] |
| Operational | Process efficiency, cycle time reduction | Before/after metrics [METRIC] |
| Strategic | Market positioning, competitive advantage | OKR achievement [PLAN] |
| Compliance | Regulatory adherence, risk reduction | Audit pass rate [METRIC] |
| Customer | Satisfaction, retention, NPS | Survey scores [STAKEHOLDER] |

## 2. Benefits Lifecycle

```
Identify → Analyze → Plan → Realize → Review → Optimize
```

### 2.1 Identification Techniques

1. **Benefits mapping** — Connect deliverables to outcomes to benefits to strategic objectives [PLAN]
2. **Stakeholder workshops** — Facilitated sessions to surface expected benefits [STAKEHOLDER]
3. **Business case review** — Extract benefits from investment justification [DOC]
4. **Value stream analysis** — Map value flow from initiative to customer impact [METRIC]

### 2.2 Benefits Map Structure

```
Enabler → Business Change → Benefit → Strategic Objective
(IT System) → (New Process) → (Reduced Cycle Time) → (Market Leadership)
```

## 3. Measurement Framework

| Metric Type | When to Measure | Method |
|-------------|----------------|--------|
| Baseline | Before project starts | Current state measurement [METRIC] |
| In-progress | During delivery | Leading indicators [SCHEDULE] |
| Post-delivery | 3-6 months after go-live | Outcome measurement [METRIC] |
| Sustained | 12-24 months | Longitudinal tracking [PLAN] |

### 3.1 KPI Design for Benefits

Each benefit requires:
- **Metric name** and calculation formula [METRIC]
- **Baseline value** (pre-project) [METRIC]
- **Target value** (expected post-project) [PLAN]
- **Measurement frequency** and data source [SCHEDULE]
- **Benefit owner** responsible for tracking [STAKEHOLDER]

## 4. Common Challenges

1. **Benefits evaporation** — Benefits claimed but never measured post-delivery [INFERENCIA]
2. **Attribution problem** — Multiple factors influence outcomes; hard to isolate project impact [SUPUESTO]
3. **Delayed realization** — Benefits materialize long after project closure [SCHEDULE]
4. **Optimism bias** — Overstating expected benefits in business cases [INFERENCIA]
