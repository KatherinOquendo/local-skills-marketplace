# Use-Case Prompts — Kickoff Package

## Prompt 1: Full Project Kickoff Package

**When**: Preparing for a new project launch with all stakeholders.

**Context variables**:
- `{PROJECT_NAME}`: Project name
- `{TEAM_SIZE}`: Number of team members
- `{STAKEHOLDER_COUNT}`: Number of stakeholders
- `{METHODOLOGY}`: Chosen delivery methodology

**Deliver**:
> "Prepare a complete kickoff package for {PROJECT_NAME} with {TEAM_SIZE} team members and {STAKEHOLDER_COUNT} stakeholders using {METHODOLOGY}. Include project charter summary, stakeholder map, team directory template, communication plan, ways of working guide, risk register, timeline, governance model, and kickoff agenda with facilitation notes."

## Prompt 2: Remote Kickoff Design

**When**: Designing a kickoff for a fully remote or distributed team.

**Context variables**:
- `{TIME_ZONES}`: Time zones involved
- `{TOOLS}`: Collaboration tools available
- `{TEAM_CULTURE}`: Team cultural context

**Deliver**:
> "Design a remote-first kickoff for a team spanning {TIME_ZONES}. Available tools: {TOOLS}. Culture: {TEAM_CULTURE}. Include async pre-work materials (videos, reading), synchronous session agenda (max 90 minutes), interactive exercises for team building, and post-kickoff follow-up plan."

## Prompt 3: Phase Kickoff Package

**When**: Re-aligning the team at the start of a new project phase.

**Context variables**:
- `{PHASE_NAME}`: New phase name
- `{PREVIOUS_PHASE_RESULTS}`: Summary of previous phase outcomes
- `{PHASE_OBJECTIVES}`: Objectives for the new phase

**Deliver**:
> "Prepare a phase kickoff package for {PHASE_NAME}. Previous phase results: {PREVIOUS_PHASE_RESULTS}. Objectives: {PHASE_OBJECTIVES}. Include lessons learned integration, updated risk register, revised team assignments, phase-specific governance, and sprint/milestone plan."
