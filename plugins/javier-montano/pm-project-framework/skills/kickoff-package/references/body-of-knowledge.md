# Body of Knowledge — Kickoff Package

## 1. Foundations of Project Kickoff

The kickoff package is the comprehensive set of materials prepared for and delivered during the project kickoff meeting. It establishes shared understanding of scope, objectives, team structure, governance, and ways of working. [DOC]

### 1.1 Kickoff Package Components

| Component | Purpose | Owner |
|-----------|---------|-------|
| Project Charter | Authorize and scope the project | Sponsor/PM |
| Stakeholder Map | Identify and classify stakeholders | PM |
| Team Directory | Contact info, roles, responsibilities | PM |
| Communication Plan | Channels, cadence, escalation | PM |
| Ways of Working | Methodology, ceremonies, tools | PM/SM |
| Risk Register (Initial) | Known risks at kickoff | PM |
| Timeline Overview | High-level milestones | PM |
| Governance Model | Decision rights, gates | PM/PMO |
| Success Criteria | Definition of project success | PM/Sponsor |

### 1.2 Kickoff Meeting Types

1. **Internal Kickoff** — Team alignment before external stakeholder engagement. [DOC]
2. **External Kickoff** — Full stakeholder kickoff with sponsor involvement. [DOC]
3. **Phase Kickoff** — Re-alignment at the start of each project phase. [PLAN]
4. **Sprint Zero Kickoff** — Technical setup and team norming sprint. [DOC]

## 2. Standards and Frameworks

### 2.1 PMBOK Project Charter

PMBOK 7th Edition positions the charter as the foundational project authorization document, central to any kickoff package. [DOC]

### 2.2 PRINCE2 Initiation Stage

PRINCE2's Initiation Stage produces the Project Initiation Document (PID), which serves as the comprehensive kickoff package. [DOC]

### 2.3 SAFe PI Planning

SAFe's PI Planning event is effectively a scaled kickoff for the next Program Increment, aligning multiple teams on objectives. [DOC]

## 3. Kickoff Agenda Design

| Segment | Duration | Content | Facilitator |
|---------|----------|---------|-------------|
| Welcome & Context | 15 min | Why this project matters | Sponsor |
| Vision & Objectives | 20 min | Success criteria, scope boundaries | PM |
| Team Introductions | 15 min | Roles, expertise, working style | PM |
| Ways of Working | 20 min | Methodology, tools, ceremonies | SM/PM |
| Governance & Decisions | 15 min | Decision rights, escalation, gates | PM |
| Timeline & Milestones | 15 min | Key dates, dependencies | PM |
| Risks & Assumptions | 15 min | Top risks, known assumptions | PM |
| Q&A & Next Steps | 15 min | Immediate actions, homework | PM |

## 4. Pre-Kickoff Checklist

| Item | Status Check | Responsible |
|------|-------------|-------------|
| Project charter signed | Approved by sponsor | PM |
| Team members confirmed | All roles staffed | Resource Manager |
| Tools provisioned | Access granted, configured | IT/DevOps |
| Meeting logistics | Room/link, recording setup | PM |
| Pre-read distributed | Sent 3+ days before | PM |
| Sponsor confirmed | Available for opening remarks | PM |

## 5. Post-Kickoff Actions

1. Distribute meeting recording and notes [PLAN]
2. Publish team directory and communication channels [DOC]
3. Confirm first sprint/phase planning date [SCHEDULE]
4. Follow up on open questions and action items [PLAN]
5. Update project plan based on kickoff discussions [DOC]

## 6. Common Pitfalls

1. **Death by PowerPoint** — Too much presentation, not enough interaction [INFERENCIA]
2. **Missing Sponsor** — Kickoff without executive presence signals low priority
3. **Scope Ambiguity** — Leaving kickoff with unclear boundaries
4. **No Follow-Through** — Great kickoff with no follow-up on action items
5. **One-Way Communication** — Not allowing team input and questions
