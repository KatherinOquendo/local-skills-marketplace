# State of the Art — Scope WBS (2024-2026)

## Trend 1: AI-Generated WBS from Requirements

AI tools parse requirements documents, user stories, and business cases to auto-generate initial WBS structures, dramatically reducing scope definition time. [DOC]

> "Principio Rector: El WBS es el ADN del proyecto — todo lo demás se deriva de él."

## Trend 2: Living WBS in Agile

WBS evolves from a static document to a living structure that decomposes progressively — near-term work packages fully detailed, future work at higher abstraction. Aligns with agile rolling-wave planning. [INFERENCIA]

## Trend 3: Visual WBS Tools

Interactive mind-map and tree-diagram tools replacing static Markdown/spreadsheet WBS. Teams collaboratively build WBS in real-time workshops with drag-and-drop decomposition. [DOC]

## Trend 4: WBS-Backlog Integration

Seamless mapping between WBS work packages and agile product backlogs (epics, features, stories). WBS provides the structural hierarchy; backlog provides the execution order. [INFERENCIA]

## Trend 5: Scope-as-Code

WBS defined in machine-readable formats (YAML, JSON) enabling automated traceability from scope to schedule, budget, and resource plans. Changes propagate automatically. [DOC]

| WBS Format | Audience | Automation |
|-----------|----------|-----------|
| Traditional (Word/Excel) | Document-centric orgs | Manual |
| Mind map (Miro/XMind) | Visual collaboration | Limited |
| YAML/JSON | Engineering teams | Full automation |
| Jira hierarchy (Epic/Story) | Agile teams | Tool-native |
