# Metaprompts — PMO Setup

> PMO-APEX Skill: `pmo-setup` · Metaprompts para adaptación de diseños de PMO
> Última actualización: 2026-03-17

---

## MP-01: PMO para Organización Ágil

```
Eres el agente de PMO setup del PMO-APEX. La organización
{nombre_organizacion} opera 100% ágil (SAFe / Scrum at Scale).
Diseña un VMO (Value Management Office) en lugar de un PMO tradicional:
1. Servicios orientados a flujo de valor, no compliance
2. Roles de coaching, no de auditoría
3. Métricas de flow (throughput, cycle time), no de adherencia
4. Governance lightweight integrada en ceremonies ágiles
Justifica cada decisión de diseño vs. PMO tradicional.
```

## MP-02: PMO para Startup en Crecimiento

```
La startup {nombre_organizacion} tiene 50 personas y está escalando
a 200 en 12 meses. No existe PMO ni prácticas PM formales.
Diseña un PMO lean que:
1. Empiece con 1 persona (PMO of One)
2. Escale servicios según crecimiento organizacional
3. No burocratice la cultura de startup
4. Demuestre valor en los primeros 30 días
Define el playbook de "PMO of One" con prioridades semanales.
```

## MP-03: PMO Federated para Multinacional

```
{nombre_organizacion} es una multinacional con presencia en {numero}
países y {numero_bus} business units. Diseña un modelo de PMO federado:
1. PMO Central: qué standards define y qué authority tiene
2. PMOs Locales: qué autonomía tienen y qué reportan
3. Coordination model entre PMO central y locales
4. Escalation paths cross-geography
5. Tool standardization vs. local flexibility
```

## MP-04: PMO Rescue (PMO en Crisis)

```
El PMO de {nombre_organizacion} fue establecido hace {meses} meses
pero está en crisis: {descripcion_crisis}. Diseña un plan de rescue:
1. Diagnóstico rápido (1 semana) de las causas raíz
2. Quick wins de supervivencia (30 días)
3. Redesign del PMO basado en lecciones aprendidas
4. Re-launch con nueva value proposition
No destruir lo que funciona — preservar y mejorar.
```

## MP-05: PMO Sunset Plan

```
La organización {nombre_organizacion} ha decidido cerrar su PMO.
Diseña un plan de sunset responsable:
1. Transición de servicios: qué se transfiere a quién
2. Knowledge transfer: documentación de procesos y lecciones
3. Impact assessment: qué se pierde al cerrar el PMO
4. Timeline de cierre gradual (no abrupto)
5. Communication plan para stakeholders afectados
```
