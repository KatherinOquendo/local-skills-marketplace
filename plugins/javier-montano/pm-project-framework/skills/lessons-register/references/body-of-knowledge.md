# Body of Knowledge — Lessons Register

## 1. Foundations of Lessons Learned Management

The lessons register is a living repository of knowledge gained from project experiences — both successes and failures. It captures, categorizes, and enables reuse of organizational learning across projects. [DOC]

### 1.1 Lessons Lifecycle

| Phase | Activity | Output |
|-------|----------|--------|
| Identification | Capture lessons during retrospectives, reviews | Raw lesson entries |
| Analysis | Root cause analysis, categorization | Analyzed lessons |
| Validation | Verify applicability and accuracy | Validated lessons |
| Storage | Index and archive in searchable repository | Lessons register |
| Dissemination | Share with relevant projects and teams | Notifications, briefings |
| Application | Apply lessons to active projects | Improved practices |

### 1.2 Lesson Categories

1. **Process** — Methodology, workflow, and ceremony improvements [DOC]
2. **Technical** — Architecture, technology, and tooling insights [DOC]
3. **People** — Team dynamics, communication, stakeholder management [STAKEHOLDER]
4. **Estimation** — Effort, duration, and cost estimation accuracy [METRIC]
5. **Risk** — Risk identification, assessment, and response effectiveness [PLAN]
6. **Vendor** — Third-party management and procurement lessons [DOC]
7. **Governance** — Decision-making, escalation, and oversight [DOC]

## 2. Standards and Frameworks

### 2.1 PMBOK Lessons Learned Register

PMBOK 7th Edition identifies the lessons learned register as a key project document, updated throughout the project and formally closed during project closure. [DOC]

### 2.2 PRINCE2 Lessons Log

PRINCE2 mandates a Lessons Log maintained throughout the project and a Lessons Report at project closure, with explicit requirements to search for prior lessons at initiation. [DOC]

### 2.3 ISO 21505 Project Governance

ISO 21505 emphasizes organizational learning from projects as a governance responsibility, linking lessons to continuous improvement. [DOC]

## 3. Lesson Entry Structure

| Field | Description | Required |
|-------|-------------|----------|
| Lesson ID | Unique identifier | Yes |
| Title | Brief description | Yes |
| Category | Process/Technical/People/etc. | Yes |
| Context | Project, phase, situation | Yes |
| Lesson Learned | What was learned | Yes |
| Root Cause | Why this happened | Yes |
| Recommendation | What to do differently | Yes |
| Impact | Quantified effect on project | Recommended |
| Applicability | Which project types benefit | Recommended |
| Status | Identified/Validated/Applied | Yes |
| Source | Retrospective, review, incident | Yes |

## 4. Collection Methods

| Method | Timing | Participants | Depth |
|--------|--------|-------------|-------|
| Sprint Retrospective | End of sprint | Team | Team-level |
| Phase Gate Review | End of phase | Stakeholders | Phase-level |
| Post-Incident Review | After incident | Involved parties | Incident-level |
| Project Closure Review | End of project | Full team | Project-level |
| Peer Review | Ad hoc | Subject matter experts | Topic-specific |
| Continuous Capture | Anytime | Anyone | Varies |

## 5. Knowledge Reuse Mechanisms

| Mechanism | Description | Effectiveness |
|-----------|-------------|--------------|
| Project Initiation Search | Mandatory review of prior lessons at kickoff | High |
| Lessons Digest | Monthly summary of new lessons distributed | Medium |
| Coaching Integration | Coaches reference lessons in mentoring | High |
| Template Embedding | Lessons embedded in project templates | Medium |
| AI-Powered Search | NLP-based relevant lesson surfacing | High |

## 6. Common Pitfalls

1. **Write-Only Repository** — Lessons captured but never read or applied [INFERENCIA]
2. **Blame Culture** — Lessons become finger-pointing exercises
3. **Vague Lessons** — "Communicate better" without actionable specifics
4. **No Root Cause** — Capturing symptoms instead of causes
5. **Project-End Only** — Waiting until closure to capture lessons
6. **No Feedback Loop** — No verification that lessons were actually applied
