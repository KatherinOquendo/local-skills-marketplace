# State of the Art — Project Charter (2024-2026)

> Current trends in project initiation and charter practices.

---

## 1. Dominant Trends

### 1.1 Lean Canvas Replacing Traditional Charters in Agile

- **What's happening:** The Lean Canvas (Ash Maurya, adapted from Business Model Canvas) is increasingly used as a charter substitute for agile projects. It fits on one page and emphasizes problem-solution fit, unfair advantage, and key metrics over traditional governance sections.
- **Impact on skill:** When `{TIPO_PROYECTO}` is Agile, offer Lean Charter variant. Include problem/solution hypothesis, key metrics, and unfair advantage alongside traditional SMART objectives.
- **Evidence:** Maurya, A. (2022). *Running Lean — 3rd Edition*; SAFe 6.0 Lean Business Case template.
- **Evidence tag:** [DOC]

### 1.2 OKR-Aligned Charters

- **What's happening:** Organizations using OKRs (Objectives and Key Results) are requiring charters to explicitly link project objectives to organizational OKRs. This creates traceability from strategy to execution and simplifies portfolio prioritization.
- **Impact on skill:** Business Case section should include explicit OKR mapping table. Each project objective should trace to ≥1 organizational Key Result.
- **Evidence:** Doerr, J. (2018). *Measure What Matters*; PMI (2024) "Strategic Alignment in Project Selection."
- **Evidence tag:** [DOC]

### 1.3 Hypothesis-Driven Charters for Innovation Projects

- **What's happening:** Innovation and R&D projects increasingly use hypothesis-driven charters instead of traditional scope statements. The charter defines testable hypotheses, validation criteria, and pivot/persevere decision points instead of fixed deliverables.
- **Impact on skill:** For Transformation project types, include hypothesis table with validation criteria. Link to `hypothesis-driven-delivery` skill for execution.
- **Evidence:** Bland, D. & Osterwalder, A. (2020). *Testing Business Ideas*; Gothelf, J. (2021). *Lean UX — 3rd Edition*.
- **Evidence tag:** [DOC]

### 1.4 Digital Charter Templates with Auto-Population

- **What's happening:** Modern PPM tools (Monday.com, Smartsheet, Planview) offer digital charter templates that auto-populate from intake forms, reducing charter creation time from days to hours. AI assistants can draft charters from project briefs.
- **Impact on skill:** Output should be structured for easy import into PPM tools. Include metadata frontmatter compatible with Jira, Azure DevOps, and Monday.com formats.
- **Evidence:** Gartner (2025) "PPM Tools Market Guide"; PMI (2025) "AI in Project Management."

## 2. Open Debates

### Charter as Living Document vs. Baseline
- PMBOK treats the charter as a fixed baseline. Agile practitioners argue it should be a living document updated as the project learns. PMO-APEX position: charter is baselined at Gate G1, but assumptions and risks are living sections updated throughout.

### Level of Detail in Charter
- Over-detailed charters delay initiation. Under-detailed charters cause misalignment. Rule of thumb: charter should answer "why, what, who, when" but NOT "how."
