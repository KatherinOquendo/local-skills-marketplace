# Metaprompts — Waterfall Methodology Assessment

> Meta-level strategies to maximize assessment accuracy and objectivity.

---

## 1. Self-Assessment Calibration

**Purpose:** Detect and correct bias when organizations self-assess their maturity.

```
For each dimension scored by stakeholder interviews:

1. INFLATION CHECK:
   - Does the self-reported score have artifact evidence?
   - If score ≥ 3.5 but no documented processes exist → flag inflation
   - Compare self-report against artifact-based scoring

2. COMPETENCY CONFUSION:
   - Does the team confuse "we know how" with "we consistently do"?
   - Knowledge ≠ practice ≠ institutionalized process
   - Score based on consistent execution, not capability

3. RECENCY BIAS:
   - Is the assessment skewed by the last project's outcome?
   - Review 3+ projects before scoring a dimension
   - Weight consistent patterns over individual results

Generate calibration adjustments. Tag [INFERENCIA] if artifact-based,
[SUPUESTO] if based on interview data alone.
```

---

## 2. Improvement ROI Prioritization

**Purpose:** Ensure improvement roadmap items are prioritized by impact, not ease.

```
For each gap in the improvement roadmap:

1. IMPACT SCORING:
   - Which project outcomes does this gap affect? (schedule, cost, quality, risk)
   - Quantify: if closed, what predictability improvement expected?
   - Score: Low (<5% improvement) / Medium (5-15%) / High (>15%)

2. EFFORT ESTIMATION:
   - FTE-months to implement the improvement
   - Organizational change required (process, tooling, culture, skills)
   - Timeline to see results (3/6/12 months)

3. DEPENDENCY MAPPING:
   - Does this improvement depend on others being completed first?
   - Are there quick wins that unlock larger improvements?

Rank by Impact/Effort ratio. Flag improvements where effort is
high but impact is [SUPUESTO] — validate before investing.
```

## 3. Progressive Disclosure Strategy

```
PHASE 1 — Always load:
- SKILL.md frontmatter + Proceso section
- Current maturity scores (if available)
- Target maturity level

PHASE 2 — On-demand:
- references/body-of-knowledge.md (maturity scale + dimensions)
  → When scoring dimensions
- Assessment methodology → When designing the assessment process

PHASE 3 — Deep:
- references/state-of-the-art.md → For hybrid maturity or data-driven assessment
- examples/sample-output.md → When formatting maturity reports
```
