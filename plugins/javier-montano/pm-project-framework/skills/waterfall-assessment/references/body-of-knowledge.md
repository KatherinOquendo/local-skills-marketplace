# Body of Knowledge — Waterfall Methodology Assessment

## 1. Definition and Scope

Waterfall methodology assessment evaluates an organization's capability to execute plan-driven (predictive) projects effectively. It measures maturity across PMBOK process groups, stage-gate discipline, documentation rigor, change control effectiveness, and earned value integration. The assessment identifies gaps between current practice and target maturity, producing a prioritized improvement roadmap. [DOC]

## 3. Five-Level Maturity Scale

| Level | Name | Characteristics | Evidence |
|:-----:|------|----------------|----------|
| 1 | Initial | Ad hoc processes, hero-dependent delivery | No templates, inconsistent outcomes [METRIC] |
| 2 | Managed | Basic planning exists per project, reactive control | Templates exist but adoption varies [PLAN] |
| 3 | Defined | Standard processes across organization, proactive | PMO standards enforced, training program [PLAN] |
| 4 | Quantitatively Managed | Metrics-driven, statistical process control | EVM baselines, SPI/CPI tracked [METRIC] |
| 5 | Optimizing | Continuous improvement, predictive analytics | Lessons learned feed process updates [HISTORICO] |

## 4. Assessment Dimensions

| Dimension | Weight | What to Assess |
|-----------|:------:|---------------|
| Process group execution | 25% | Completeness of Initiating → Closing process groups |
| Stage-gate discipline | 20% | Gate criteria definition, approval rigor, documentation |
| Planning rigor | 20% | WBS quality, schedule realism, resource planning |
| Change control | 15% | CCB existence, change request process, impact analysis |
| Earned value integration | 10% | EVM adoption, SPI/CPI tracking, forecasting |
| Lessons learned | 10% | Capture process, reuse in future projects |

## 5. Assessment Methodology

```
Step 1: Define scope — Which projects/programs to assess
Step 2: Gather evidence — Artifacts, interviews, observations
Step 3: Score dimensions — Using rubric per maturity level
Step 4: Gap analysis — Current vs. target maturity per dimension
Step 5: Root cause — Why gaps exist (culture, skill, tooling, governance)
Step 6: Improvement roadmap — Prioritized by impact and feasibility
Step 7: Baseline — Establish metrics for tracking improvement
```

## 6. Common Gaps by Maturity Level

| Transition | Typical Gaps | Intervention |
|-----------|-------------|-------------|
| L1 → L2 | No templates, no planning discipline | Template library + PM training [PLAN] |
| L2 → L3 | Inconsistent adoption across projects | PMO governance + compliance audits [PLAN] |
| L3 → L4 | No quantitative metrics | EVM implementation + dashboard [METRIC] |
| L4 → L5 | No feedback loop from lessons learned | Retrospective process + knowledge base [HISTORICO] |
