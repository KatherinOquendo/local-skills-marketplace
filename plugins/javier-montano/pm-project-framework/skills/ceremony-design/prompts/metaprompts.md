# Metaprompts — Ceremony Design

## Meta-Strategy 1: Ceremony ROI Analysis

**Purpose**: Justify ceremony time investment with measurable outcomes.

**Strategy**:
For each ceremony, calculate:
- Time cost = Participants x Duration x Frequency [METRIC]
- Value produced = Decisions made + Blockers resolved + Alignment created
- ROI = Value / Time cost

**Application template**:
> Calculate ceremony ROI for `{team}`. If any ceremony costs >5% of sprint capacity and produces <2 actionable outcomes per instance, flag for redesign or elimination. Present as cost-benefit table to team for discussion. [METRIC]

## Meta-Strategy 2: Format Rotation Protocol

**Purpose**: Prevent ceremony staleness through systematic format variation.

**Strategy**:
Rotate ceremony formats on a 4-sprint cycle:
- Sprint 1: Standard format (baseline)
- Sprint 2: Liberating Structures variation
- Sprint 3: Async-first with sync debrief
- Sprint 4: Team-chosen experimental format

**Application template**:
> For `{ceremony}` in `{team}`, implement 4-sprint rotation. Track satisfaction and output quality per format. After 2 full rotations (8 sprints), let team vote on preferred formats. Document what works in ceremony playbook. [STAKEHOLDER]

## Meta-Strategy 3: Inclusive Design Check

**Purpose**: Ensure ceremonies work for all personality types and work modes.

**Strategy**:
For each ceremony, verify:
- Introverts have written/async input channels
- Remote participants have equal voice (not afterthought)
- Non-native speakers have processing time built in
- Neurodivergent team members have agenda and structure in advance

**Application template**:
> Review `{ceremony_design}` through inclusion lens. Score each of 4 inclusion criteria (1-5). For any score <3, modify the format to address the gap. Document accommodations as standard ceremony protocol, not exceptions. [STAKEHOLDER]
