# Body of Knowledge — Retrospective Engine

## 1. Definition and Scope

The retrospective engine is a structured process for facilitating team reflection on completed work periods (sprints, phases, releases) to identify improvements, celebrate successes, and drive continuous process evolution. It goes beyond simple "what went well/poorly" to produce actionable, measurable improvement experiments. [DOC]

## 3. Retrospective Formats

| Format | Duration | Best For | Structure |
|--------|----------|----------|-----------|
| Standard (Start/Stop/Continue) | 60 min | General sprints | 3 columns brainstorm |
| 4Ls (Liked/Learned/Lacked/Longed-for) | 60 min | Team learning focus | 4 quadrant |
| Sailboat | 45 min | Blockers identification | Wind/anchor metaphor |
| Timeline | 90 min | Release/phase retrospective | Chronological events |
| Fishbone/Ishikawa | 60 min | Root cause analysis | Cause-effect diagram |
| Lean Coffee | 45 min | Open agenda | Dot-voted topics |

## 4. Five-Stage Process

1. **Set the Stage** — Create psychological safety, state purpose and ground rules
2. **Gather Data** — Collect observations, metrics, and experiences from the period
3. **Generate Insights** — Analyze patterns, identify root causes, find connections
4. **Decide Actions** — Select improvement experiments with owners and due dates
5. **Close** — Appreciation round, feedback on the retro itself [PLAN]

## 5. Quantitative Retrospective Elements

| Metric | Source | Purpose |
|--------|--------|---------|
| Velocity trend | Sprint data | Productivity trajectory [METRIC] |
| Defect injection rate | QA data | Quality trajectory |
| Cycle time distribution | Board data | Flow health |
| Team happiness index | Survey | Engagement trajectory |
| Action item completion rate | Previous retro | Improvement effectiveness |

## 6. Anti-Patterns

| Anti-Pattern | Symptom | Remedy |
|-------------|---------|--------|
| Groundhog Day retro | Same issues every sprint | Track action items, measure resolution |
| Blame game | Personal attacks instead of system thinking | Establish prime directive, facilitate safety |
| Parking lot | Actions never implemented | Assign owners, track in sprint backlog |
| Skip-it culture | Retros cancelled for "real work" | Non-negotiable timebox, exec sponsorship |
| Recency bias | Only discussing last 2 days | Use data and timeline exercises |
