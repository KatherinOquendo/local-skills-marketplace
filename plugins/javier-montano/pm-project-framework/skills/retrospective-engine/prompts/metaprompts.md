# Metaprompts — Retrospective Engine

## Meta-Strategy 1: Psychological Safety Assurance

**Purpose:** Ensure retro environment enables honest reflection

```
Before generating retro outputs, verify:
1. PRIME DIRECTIVE stated: "Regardless of what we discover, we understand that everyone did the best job they could, given what they knew at the time."
2. Observations are about SYSTEMS and PROCESSES, not people
3. Language is improvement-oriented, not blame-oriented
4. Minority opinions are actively sought (What did we NOT hear?)
5. Vulnerability is modeled (facilitator shares own learning first)
Rewrite any output that violates these principles.
```

## Meta-Strategy 2: Action Quality Gate

**Purpose:** Ensure retro actions are actionable and trackable

```
For each proposed action item, verify:
1. SPECIFIC — Is the action concrete enough to execute? ("Improve testing" → fail)
2. OWNED — Is there a single owner? (Shared ownership = no ownership)
3. TIME-BOXED — Is there a deadline? (Default: by next retro)
4. MEASURABLE — How will we know it worked? (Define success metric)
5. SCOPED — Can it be completed within 1-2 sprints? (Break down if larger)
Reject actions that fail 2+ criteria. Rewrite them.
```

## Meta-Strategy 3: Pattern Detection Across Retros

**Purpose:** Identify recurring themes that require systemic intervention

```
Analyze the last 3-5 retrospective records for patterns:
1. RECURRING THEMES — What topics appear in >50% of retros? These need systemic solutions.
2. STALE ACTIONS — What actions were created but never completed? Why?
3. IMPROVEMENT VELOCITY — Is the team actually improving? (Compare metrics across retros)
4. BLIND SPOTS — What topics are NEVER discussed? (Quality? Dependencies? Communication?)
Produce a retrospective meta-analysis with systemic recommendations.
```
