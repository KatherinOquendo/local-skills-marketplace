## 1. Fundamentos Teóricos

### 1.1 Portfolio Management vs. Project Management
Portfolio management opera a nivel estratégico: selecciona los proyectos correctos, balancea inversión vs. riesgo, y maximiza valor organizacional. Project management opera a nivel táctico: ejecuta proyectos individuales correctamente [DOC].

### 1.2 Dimensiones de Evaluación (6 Pilares)
| # | Dimensión | Pregunta Clave |
|---|-----------|---------------|
| 1 | Strategic Alignment | Los proyectos conectan con la estrategia? |
| 2 | Portfolio Governance | Hay un cuerpo que toma decisiones de portafolio? |
| 3 | Prioritization | Se usa un modelo de scoring transparente? |
| 4 | Resource Optimization | Se optimizan recursos cross-project? |
| 5 | Risk Aggregation | Se analizan riesgos a nivel de portafolio? |
| 6 | Value Realization | Se mide el valor entregado por el portafolio? |

### 1.3 Madurez de Portfolio Management
| Nivel | Descripción |
|-------|-------------|
| 1 | Project list — sin priorización formal |
| 2 | Governance básica — portfolio board existe pero es reactivo |
| 3 | Portfolio management — scoring, priorización, y governance activos |
| 4 | Portfolio optimization — efficient frontier, resource balancing |
| 5 | Strategic portfolio — investment management con feedback loops |

## 2. Marcos de Referencia

### 2.1 PMI Standard for Portfolio Management
Define portfolio management como alineación de componentes (proyectos, programas) con estrategia organizacional. Incluye: Portfolio Strategic Management, Portfolio Governance, Portfolio Performance, Portfolio Communication, Portfolio Risk [DOC].

### 2.2 MoP (Management of Portfolios)
Framework de Axelos para portfolio management. Enfoca en portfolio definition (selección) y portfolio delivery (ejecución). Incluye governance structure, prioritization, y benefits management [DOC].

### 2.3 SAFe Lean Portfolio Management
Framework ágil para portfolio management: Strategy and Investment Funding, Agile Portfolio Operations, Lean Governance. Usa portfolio Kanban y guardrails en lugar de approval gates [DOC].

## 3. Métodos de Assessment

### 3.1 Document Review
Revisar documentación de governance, scoring models, portfolio reports, y minutes de portfolio board para evidencia de prácticas formales [PLAN].

### 3.2 Stakeholder Interviews
Entrevistar portfolio board members, sponsors, PMs, y ejecutivos para evaluar percepción de efectividad de la gestión de portafolio [STAKEHOLDER].

### 3.3 Data Analysis
Analizar datos de portafolio: strategic alignment scores, project selection rationale, resource allocation patterns, y portfolio performance metrics [METRIC].

## 4. Gap Analysis Framework

Para cada dimensión: Score actual (1-5) vs. Score target = Gap
Priorizar gaps por: impacto estratégico x factibilidad de mejora
Diseñar roadmap de mejora por dimensión con timeline y esfuerzo estimado [PLAN].

## 5. Anti-patrones

| Anti-patrón | Consecuencia | Corrección |
|-------------|-------------|------------|
| Portfolio = project list | Sin priorización ni governance | Establecer portfolio board |
| HiPPO prioritization | Decisiones políticas, no estratégicas | Scoring model transparente |
| Over-committed portfolio | Todos los proyectos aprobados, ninguno terminado | WIP limits a nivel portfolio |
| No benefits tracking | Sin evidencia de valor entregado | Benefits realization process |
| Siloed project execution | Sin optimización cross-project | Resource pooling y visibility |
