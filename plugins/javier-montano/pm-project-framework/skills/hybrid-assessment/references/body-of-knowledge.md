# Body of Knowledge — Hybrid Assessment

## 1. Foundations of Hybrid Assessment

Hybrid assessment evaluates an organization's readiness and capability to blend predictive (waterfall) and adaptive (agile) approaches within a unified delivery framework. It determines the optimal methodology mix for project context. [DOC]

### 1.1 Assessment Dimensions

| Dimension | Description | Key Indicators |
|-----------|-------------|----------------|
| Organizational Culture | Tolerance for ambiguity, change appetite | Leadership style, decision speed |
| Team Maturity | Agile/predictive experience levels | Certifications, retrospective quality |
| Requirements Clarity | Degree of upfront definition possible | Stakeholder availability, domain novelty |
| Technical Complexity | Architecture constraints, integration needs | Coupling, legacy dependencies |
| Regulatory Environment | Compliance documentation requirements | Audit frequency, mandatory artifacts |
| Stakeholder Expectations | Governance and reporting preferences | Meeting cadence, artifact expectations |

### 1.2 Hybrid Maturity Levels

1. **Ad-hoc** — Teams choose methods informally with no organizational guidance. [INFERENCIA]
2. **Emerging** — Some projects use hybrid but without standardized approach. [DOC]
3. **Defined** — Organization has a hybrid methodology guide with decision criteria. [DOC]
4. **Managed** — Metrics track hybrid effectiveness; continuous calibration. [PLAN]
5. **Optimized** — AI-assisted methodology selection; real-time adaptation. [INFERENCIA]

## 2. Standards and Frameworks

### 2.1 PMI Disciplined Agile (DA)

DA provides a process decision framework with goal-driven lifecycles: Agile, Lean, Continuous Delivery, Exploratory, and Program. The "Choose Your WoW" approach is inherently hybrid. [DOC]

### 2.2 PRINCE2 Agile

PRINCE2 Agile combines PRINCE2 governance with agile delivery techniques, using an Agilometer to assess suitability of agile practices per project dimension. [DOC]

### 2.3 SAFe Configuration

SAFe offers Essential, Large Solution, Portfolio, and Full configurations, each blending agile team practices with enterprise planning and governance. [DOC]

## 4. Decision Criteria for Methodology Selection

| Factor | Favors Predictive | Favors Adaptive | Hybrid Approach |
|--------|-------------------|-----------------|-----------------|
| Requirements | Well-defined, stable | Emergent, evolving | Stable core + adaptive features |
| Team Size | Large, distributed | Small, co-located | Core team agile + extended predictive |
| Compliance | Heavy regulation | Light regulation | Agile delivery + predictive governance |
| Stakeholders | Low availability | High availability | Sprint demos + phase gates |
| Technology | Proven, stable | Novel, uncertain | Architecture predictive + dev agile |

## 5. Assessment Process

1. **Scoping** — Define assessment boundaries and stakeholders [PLAN]
2. **Data Collection** — Surveys, interviews, artifact review [METRIC]
3. **Analysis** — Map findings to maturity model [INFERENCIA]
4. **Recommendations** — Tailored hybrid configuration [PLAN]
5. **Roadmap** — Transition plan with milestones [SCHEDULE]

## 6. Common Pitfalls

1. **Water-Scrum-Fall** — Agile in name only; waterfall gates remain unchanged [INFERENCIA]
2. **Tool-First Thinking** — Selecting Jira before defining methodology
3. **One Assessment Fits All** — Not differentiating by project type
4. **Ignoring Culture** — Imposing hybrid on resistant organizations
5. **Assessment Fatigue** — Over-assessing without action on findings
