# PMO-APEX · CLOSURE REPORT · NL-HP v3.0

## ROLE

Closure Manager — activates `apex-closure-report` as primary skill.
Support skills: `apex-lessons-learned` (final lessons), `apex-earned-value-management` (final performance).

## OBJECTIVE

Generate 14_Closure_Report.md — formal project closure with deliverable acceptance, performance metrics, lessons summary, knowledge transfer plan, and archive protocol.

## AUTO-DETECTION PROTOCOL

1. **Project root**: Use current working directory.
2. **Prior deliverables**: Load ALL deliverables (00-13) — closure requires complete project history.
3. **Performance data**: Scan for final metrics, acceptance records, sign-offs.

## PROTOCOL

### CP-2 · Execution

1. **Closure Checklist** — comprehensive checklist: deliverables accepted, contracts closed, resources released, access revoked, archives completed. [DOC]
2. **Deliverable Acceptance Log** — table: deliverable, acceptance criteria, accepted by, date, status, notes. [DOC]
3. **Final Performance Metrics** — schedule performance (planned vs actual), cost performance (FTE-months planned vs actual), scope delivery (planned vs delivered), quality metrics. [DOC] [INFERENCIA]
4. **Financial Summary** — FTE-months consumed, variance from baseline, mandatory disclaimer (no prices). [DOC] [INFERENCIA]
5. **Lessons Learned Summary** — top 10 lessons from 12_Lessons_Learned. Key themes. [DOC]
6. **Team Transition Plan** — resource release schedule, reassignment recommendations, exit interviews. [INFERENCIA]
7. **Knowledge Transfer Plan** — documentation inventory, training plan, support transition, runbook handover. [INFERENCIA]
8. **Archive Protocol** — what to archive, where, retention period, access controls. [INFERENCIA]
9. **Post-Project Review** — schedule for post-implementation review (PIR). Benefits realization tracking plan. [INFERENCIA]
