## AUTO-DETECTION PROTOCOL

2. **DevOps signals**: Scan for CI/CD pipelines, infrastructure-as-code, monitoring config, deployment scripts.
3. **PM signals**: Scan for project plans, backlogs, release plans.

## PROTOCOL

1. **DevOps Landscape** — CI/CD maturity, deployment frequency, infrastructure automation, monitoring coverage. [CODIGO] [CONFIG] [INFERENCIA]
2. **PM-DevOps Integration** — how PM practices align with DevOps: release planning, change management, incident management, capacity planning. [INFERENCIA]
3. **Flow Analysis** — value stream from idea to production: handoffs, wait times, automation gaps. [INFERENCIA]
4. **DORA Metrics** — deployment frequency, lead time for changes, change failure rate, mean time to recovery. [INFERENCIA]
5. **Friction Points** — where PM processes slow DevOps, where DevOps creates PM blind spots. [INFERENCIA]
6. **Recommendations** — alignment improvements: automate PM data collection, integrate dashboards, align ceremonies. [INFERENCIA]

## OUTPUT CONFIGURATION

- **Language**: Spanish (Latin American)

## CONSTRAINTS

- DevOps and PM must serve the same goal — value delivery speed and quality.
- Do not recommend processes that slow down deployment frequency.
- DORA metrics are the north star for DevOps health.
