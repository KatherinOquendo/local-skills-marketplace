# PMO Models & Governance — Core Knowledge Primer

> **Propósito RAG**: Este documento provee conocimiento de dominio para enriquecer el contexto del agente durante sesiones de gerencia de proyectos. No es un entregable — es material de referencia interno.

## PMO Types

### Three Classical PMO Models (PMI)

| Type | Authority Level | Services | Decision Rights |
|------|---------------|----------|----------------|
| Supportive | Low | Templates, training, mentoring, best practices | Advisory only; PMs decide |
| Controlling | Medium | Compliance audits, methodology enforcement, reporting | Require conformance; PMs execute within guidelines |
| Directive | High | Direct project management, PM assignment, resource allocation | PMO manages projects directly |

### Extended PMO Models

| Model | Scope | Focus |
|-------|-------|-------|
| Project PMO | Single large project | Governance and support for one project |
| Program PMO | Program of related projects | Benefits coordination, inter-project dependencies |
| Portfolio PMO (EPMO) | Enterprise-wide | Strategic alignment, resource optimization |
| Center of Excellence (CoE) | Practice development | Capability building, methodology evolution |
| Agile PMO (APMO) | Agile transformation | Lean governance, agile coaching, value streams |
| Value Management Office (VMO) | Outcomes focus | Shift from delivery tracking to value measurement |

### PMO Evolution Path
```
No PMO → Supportive PMO → Controlling PMO → Directive PMO → EPMO/VMO
          (templates)       (standards)       (management)    (strategy)
```

### PMO Services Catalog

| Service Category | Supportive | Controlling | Directive |
|-----------------|-----------|-------------|-----------|
| Templates & tools | Provide | Mandate | Manage |
| Methodology | Recommend | Enforce | Own |
| Training & coaching | Offer | Require | Deliver |
| Resource management | Track | Coordinate | Allocate |
| Reporting | Aggregate | Standardize | Own |
| Risk management | Advise | Audit | Manage |
| PM assignment | Suggest | Approve | Assign |
| Vendor management | Support | Oversee | Manage |
| Benefits tracking | Optional | Required | Owned |

## PMO Maturity Models

### OPM3 (Organizational Project Management Maturity Model — PMI)

| Domain | Scope |
|--------|-------|
| Project Management | Individual project practices |
| Program Management | Related project coordination |
| Portfolio Management | Strategic alignment, investment decisions |

#### OPM3 Process Improvement Stages

| Stage | Focus | Evidence |
|-------|-------|---------|
| Standardize | Documented processes exist | Written procedures, templates |
| Measure | Metrics collected, performance tracked | KPIs defined and reported |
| Control | Processes managed within limits | Variance triggers corrective action |
| Continuously Improve | Systematic optimization | Retrospectives drive measurable improvement |

### P3M3 (Portfolio, Programme and Project Management Maturity Model)

| Level | Name | Description |
|-------|------|------------|
| 1 | Awareness of process | Ad hoc, inconsistent, hero-dependent |
| 2 | Repeatable process | Basic processes exist, applied inconsistently |
| 3 | Defined process | Documented, standardized, organization-wide |
| 4 | Managed process | Quantitatively measured and controlled |
| 5 | Optimized process | Continuous improvement, innovation embedded |

#### P3M3 Perspectives (assessed at each level)

| Perspective | Focus Area |
|------------|-----------|
| Management Control | Governance, decision-making, escalation |
| Benefits Management | Benefits identification, tracking, realization |
| Financial Management | Budgeting, forecasting, cost control |
| Stakeholder Management | Engagement, communication, expectations |
| Risk Management | Identification, assessment, response, monitoring |
| Organizational Governance | Alignment with corporate governance |
| Resource Management | Capacity planning, allocation, optimization |

### Gartner PPM Maturity Model

| Level | Name | Characteristics |
|-------|------|----------------|
| 1 | Reactive | No standardization, success is luck-dependent |
| 2 | Emerging Discipline | Basic PMO, some methodology, limited portfolio view |
| 3 | Initial Integration | Portfolio processes defined, resource management emerging |
| 4 | Effective Integration | Portfolio optimization, demand management, benefits tracking |
| 5 | Effective Innovation | Adaptive governance, continuous optimization, strategic PMO |

### Maturity Assessment Decision Guide

| Organization Need | Recommended Model |
|-------------------|------------------|
| PMI-aligned organization | OPM3 |
| UK government / PRINCE2 environment | P3M3 |
| Executive-level maturity conversation | Gartner PPM |
| Agile-specific maturity | Agile Fluency Model or SAFe Business Agility Assessment |

## Portfolio Management (PPM)

### Portfolio Management Process
```
Strategic Planning → Identification → Evaluation → Balancing & Authorization
        ↑                                              ↓
Portfolio Review ←──────── Monitoring ←────────────────┘
```

### Portfolio Components

| Component | Definition | Key Difference |
|-----------|-----------|---------------|
| Project | Temporary, unique deliverable | Defined start/end, specific scope |
| Program | Group of related projects | Coordinated benefits, shared resources |
| Operations | Ongoing, repetitive work | No end date, steady state |
| Portfolio | Collection of projects, programs, operations | Aligned to strategic objectives |

### Portfolio Balancing Dimensions

| Dimension | Balance Between |
|-----------|----------------|
| Risk | Conservative vs. aggressive investments |
| Timeline | Short-term wins vs. long-term strategic bets |
| Innovation | Run/Grow/Transform (Horizon 1/2/3) |
| Business area | Distribution across departments/BUs |
| Resource type | In-house vs. outsourced |
| Investment type | Mandatory (compliance) vs. discretionary (growth) |

### Run/Grow/Transform Model

| Category | Purpose | Typical Allocation |
|----------|---------|-------------------|
| Run | Keep the lights on, maintenance, compliance | 50-70% |
| Grow | Enhance existing products/services | 20-30% |
| Transform | New business models, disruptive innovation | 10-20% |

## Program Management

### Program vs. Project

| Aspect | Project | Program |
|--------|---------|---------|
| Focus | Deliverables (outputs) | Benefits (outcomes) |
| Scope | Defined, bounded | Evolving, strategic |
| Duration | Temporary, defined end | Longer-term, phased |
| Change | Managed through change control | Expected and embraced |
| Success metric | On time, budget, scope | Benefits realized |
| Manager role | Tactical execution | Strategic coordination |

### Program Management Performance Domains (PMI)

| Domain | Focus |
|--------|-------|
| Strategy Alignment | Link program to organizational strategy |
| Benefits Management | Identify, plan, deliver, sustain benefits |
| Stakeholder Engagement | Manage stakeholders across projects |
| Governance | Decision structures, phase gates, oversight |
| Life Cycle Management | Initiation, delivery, closure |

### Benefits Management Lifecycle
```
Identify → Plan → Deliver → Sustain → Review
```

| Phase | Activities | Key Outputs |
|-------|-----------|-------------|
| Identify | Map benefits to strategy, quantify | Benefits register, benefits map |
| Plan | Define KPIs, baselines, methods, owners | Benefits realization plan |
| Deliver | Execute projects, track leading indicators | Benefits progress reports |
| Sustain | Transition to operations, ongoing measurement | Sustained benefits confirmation |
| Review | Post-program assessment | Benefits review report, lessons |

## Governance Frameworks

### Project Governance Structure
```
Board of Directors / Executive Committee
        ↓
Portfolio Governance Board (strategic alignment, investment decisions)
        ↓
Program Steering Committee (benefits oversight, inter-project coordination)
        ↓
Project Board / Steering Committee (project direction, stage gates)
        ↓
Project Manager (day-to-day execution within delegated authority)
        ↓
Team Leads (delivery within work package tolerances)
```

### Governance Decision Rights

| Decision Type | Decision Maker | Input From |
|--------------|---------------|-----------|
| Strategic direction | Portfolio board | Strategy office, PMO |
| Project authorization | Portfolio board | PMO, business case owner |
| Stage/phase gate | Project board | PM, project assurance |
| Change requests (major) | Project board | PM, change authority |
| Change requests (minor) | Change authority / PM | Team, stakeholders |
| Cross-project resources | Portfolio PMO | PMs, resource managers |
| Risk escalation | Project board | PM, risk owner |
| Benefits realization | Program board | Program manager, benefit owners |

### Quality Gates (Stage Gates)

| Gate | Decision Options | Criteria |
|------|-----------------|---------|
| G0 | Concept approved / rejected | Strategic alignment, feasibility |
| G1 | Charter approved | Business case, scope, high-level plan |
| G2 | Baseline approved, execution authorized | Detailed plan, budget, risk analysis |
| G3 | Deliverable accepted | Quality criteria met, stakeholder acceptance |
| G4 | Project closed, benefits transition | Lessons captured, operations handover |

## Project Selection Methods

### Scoring Models

| Step | Activity |
|------|---------|
| 1 | Define criteria (strategic fit, risk, ROI, feasibility, urgency) |
| 2 | Assign weights (total = 100%) |
| 3 | Score each project (1-10 scale) |
| 4 | Calculate weighted score: Σ (weight x score) |
| 5 | Rank projects by weighted score |

#### Example Scoring Matrix

| Criterion | Weight | Project A | Project B | Project C |
|-----------|--------|-----------|-----------|-----------|
| Strategic alignment | 30% | 8 (2.4) | 6 (1.8) | 9 (2.7) |
| Financial return | 25% | 7 (1.75) | 9 (2.25) | 5 (1.25) |
| Technical feasibility | 20% | 9 (1.8) | 5 (1.0) | 7 (1.4) |
| Risk level (inverted) | 15% | 6 (0.9) | 8 (1.2) | 4 (0.6) |
| Resource availability | 10% | 7 (0.7) | 7 (0.7) | 8 (0.8) |
| **Weighted Total** | **100%** | **7.55** | **6.95** | **6.75** |

### Analytic Hierarchy Process (AHP)

| Step | Activity |
|------|---------|
| 1 | Define criteria hierarchy |
| 2 | Pairwise comparison (1-9 scale) |
| 3 | Calculate priority weights |
| 4 | Check consistency ratio (CR < 0.10) |
| 5 | Compare alternatives against each criterion |
| 6 | Synthesize overall priority scores |

#### AHP Scale

| Value | Meaning |
|-------|---------|
| 1 | Equal importance |
| 3 | Moderate importance of one over another |
| 5 | Strong importance |
| 7 | Very strong importance |
| 9 | Extreme importance |
| 2,4,6,8 | Intermediate values |

### Other Selection Methods

| Method | Description | Best For |
|--------|------------|----------|
| Cost-Benefit Analysis | NPV, IRR, payback, BCR | Financial justification |
| Pairwise Comparison | Compare each project against every other | Small project lists |
| Bubble Chart | 2 axes + bubble size | Visual portfolio balancing |
| Constrained Optimization | Linear programming | Mathematical optimization |
| Real Options | Projects as options | High-uncertainty, staged investments |

### Financial Selection Metrics

| Metric | Formula | Decision Rule |
|--------|---------|--------------|
| NPV | Σ [Cash Flow / (1+r)^t] | Higher NPV preferred; >0 to proceed |
| IRR | Rate where NPV = 0 | IRR > hurdle rate |
| Payback Period | Time to recover investment | Shorter preferred |
| BCR | PV(Benefits) / PV(Costs) | >1.0 to proceed |
| ROI | (Net Benefits / Costs) x 100 | Higher preferred |

## Benefits Realization Tracking

### Benefits Register Fields

| Field | Content |
|-------|---------|
| Benefit ID | Unique identifier |
| Description | What the benefit is |
| Category | Financial, operational, strategic, compliance |
| Owner | Person accountable for realization |
| KPI | Measurement method |
| Baseline | Current state |
| Target | Expected improvement |
| Timeline | When benefit materializes |
| Dependencies | Prerequisites |
| Status | Not started / In progress / Realized / Not realized |

### Benefits Categories

| Category | Examples | Measurement |
|----------|---------|------------|
| Financial | Revenue increase, cost reduction | Dollar value, % change |
| Operational | Process efficiency, cycle time | Time saved, throughput |
| Strategic | Market position, competitive advantage | Market share, NPS |
| Compliance | Regulatory adherence, risk reduction | Audit results, incidents |
| Social/Environmental | Sustainability, satisfaction | ESG scores, surveys |

### Benefits Realization Challenges

| Challenge | Mitigation |
|-----------|-----------|
| Benefits too vague | Define SMART KPIs at initiation |
| No baseline | Measure current state before project starts |
| Ownership unclear | Assign benefit owners (not PM) at initiation |
| Measured too early | Allow transition period; measure at planned intervals |
| No post-project follow-up | Schedule reviews at 3, 6, 12 months |
| Attribution errors | Control for external factors, use comparison groups |

## PMO Value Proposition

### PMO Value Metrics

| Category | Metric | Target |
|----------|--------|--------|
| Delivery | Project success rate | >75% |
| Delivery | Benefits realization rate | >80% |
| Financial | Portfolio ROI | Organization-specific |
| Financial | Resource utilization | 75-85% (not 100%) |
| Capability | PM competency improvement | YoY improvement |
| Strategic | Projects aligned to strategy | >90% |
| Governance | Decision cycle time | Decreasing |
| Satisfaction | Stakeholder NPS for PMO | >30 |

### PMO Anti-Patterns

| Anti-Pattern | Symptom | Fix |
|-------------|---------|-----|
| Bureaucracy factory | Process overhead > value | Tailor governance to project size/risk |
| Report collector | Data nobody uses | Align reporting to decisions |
| Ivory tower | Disconnected from reality | Embed PMO in projects, PM rotation |
| Template police | Format over content | Focus on outcomes |
| Scope creep PMO | Takes on non-PM functions | Define charter, review annually |

## Quick Reference

| Element | Key Principle | Decision Criteria |
|---------|-------------|------------------|
| PMO type | Match authority to culture | Start supportive, grow as trust builds |
| Maturity model | Assess before prescribing | Choose model matching context |
| Portfolio balance | Align to strategy, diversify | Run/Grow/Transform allocation |
| Program vs. project | Programs = benefits, projects = outputs | Benefits coordination needed? → program |
| Project selection | Transparent, criteria-based | Weighted scoring + financial analysis |
| Governance | Appropriate to size and risk | Light for small, heavy for critical |
| Benefits tracking | Assign owners, measure baselines | Start measurement before execution |
