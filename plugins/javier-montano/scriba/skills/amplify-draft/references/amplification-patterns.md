# Amplification Patterns — Strategies and Tagging Conventions

## Core Principle: Additive Only

Amplification is strictly additive. The human author's content is sacred and must never be modified, deleted, or reworded. All interventions are additions, annotations, or suggestions.

## Tagging Convention

| Tag | Meaning | Applied To |
|-----|---------|------------|
| [ORIGINAL] | Content written by the human author | All original text (implicit — no tag needed in output) |
| [GENERADO] | Content generated by Scriba agents | New sections, DUA adaptations, gap-fills |
| [REF: DZ-xxx] | Curriculum traceability reference | Every paragraph linked to a destreza |
| [REF: PENDIENTE] | No matching destreza found | Paragraphs needing human review for DZ mapping |
| [BLOOM: nivel] | Taxonomic annotation (Bloom) | Activities and questions |
| [MARZANO: nivel] | Taxonomic annotation (Marzano) | Activities and questions |
| [SUGERENCIA] | Style correction suggestion | Flagged text needing human approval |
| [SUPUESTO] | Inference not directly from source | Any assumption made during amplification |

## Amplification Passes (Sequential)

### Pass 1: Structural Analysis
Map the draft against the Sinapsis unit model:

```
Sinapsis Model:
1. Apertura
   - Experiencia Concreta (actividad motivadora)
   - Mi emocion cuenta (conexion emocional)
2. Desarrollo
   - Conceptualizacion (contenido teorico)
   - Piensa y Reflexiona (metacognicion)
   - Actividades progresivas (individual → grupal → cooperativa)
3. Cierre
   - Ponte pilas (aplicacion practica)
   - Reflexion final
4. Evaluacion
   - Rubrica con indicadores
   - Autoevaluacion
```

For each section: mark as PRESENT, PARTIAL, or ABSENT.

### Pass 2: Traceability (DZ Tags)
For each content paragraph:
1. Identify the core concept or skill being developed
2. Match against the curriculum matrix (asignatura + grado)
3. Add [REF: DZ-xxx] tag
4. If no match: add [REF: PENDIENTE]

Rules:
- One paragraph may map to multiple destrezas — use the primary one
- Never force a mapping — PENDIENTE is preferable to incorrect mapping
- Cross-reference with the matriz A&S if available

### Pass 3: DUA Adaptations
For each activity:
1. Check if DUA adaptations exist (minimum 2 per activity)
2. If missing, generate adaptations covering at least 2 of 3 principles:
   - Principle 1: Multiple means of representation (visual, auditory, tactile)
   - Principle 2: Multiple means of action and expression (written, oral, digital)
   - Principle 3: Multiple means of engagement (choice, relevance, self-regulation)
3. Mark generated adaptations with [GENERADO]
4. Place after the original activity, clearly separated

### Pass 4: Style Review
Generate a changelog (NOT auto-applied):

| Check | Threshold | Action |
|-------|-----------|--------|
| Sentence length | >20 words | Flag with [SUGERENCIA]: suggest split |
| Vocabulary level | Above grade | Flag with [SUGERENCIA]: suggest simpler synonym |
| Tone | Not matching Manual de Voz | Flag with [SUGERENCIA]: cite rule |
| Passive voice | >30% of sentences | Flag with [SUGERENCIA]: suggest active |
| Gender-inclusive language | Missing | Flag with [SUGERENCIA]: suggest inclusive form |

### Pass 5: Gap Filling
For each ABSENT section in the Sinapsis model:
1. Generate the missing section content
2. Mark entirely with [GENERADO]
3. Place in the correct position within the unit structure
4. Ensure generated content connects logically with existing [ORIGINAL] content

## Diff Report Structure

The amplification report must clearly separate:

```markdown
## Contenido Original Preservado
- Total de palabras originales: N
- Porcentaje preservado: 100%
- Secciones originales: {list}

## Adiciones [GENERADO]
- Secciones nuevas: N ({list})
- Adaptaciones DUA: N
- Total palabras generadas: N

## Anotaciones
- Tags DZ anadidos: N
- Tags PENDIENTE: N (requieren revision humana)
- Anotaciones Bloom/Marzano: N

## Sugerencias de Estilo [SUGERENCIA]
- Total sugerencias: N
- Estado: PENDIENTE aprobacion del autor
```

## Collaboration Protocol with Human Author

```
Author v1 ──[amplify-draft]──> v1.5 (annotated)
                                  │
                            Author reviews
                                  │
                            Accepts/Rejects
                                  │
                              Author v2 ──> Pipeline continues
```

The author always has final authority. No [SUGERENCIA] is applied without explicit approval.
