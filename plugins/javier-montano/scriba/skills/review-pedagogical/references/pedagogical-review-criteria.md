# Pedagogical Review Criteria

> Domain knowledge for the `review-pedagogical` skill.

## Backward Design Checklist

Backward design (Wiggins & McTighe) requires verifying a three-link chain for every learning objective in the unit.

### Verification Steps

| Step | Check | Pass Criteria | Failure Mode |
|------|-------|--------------|-------------|
| 1. Objectives exist | Every destreza has a stated learning objective | 1:1 mapping | Missing objectives |
| 2. Assessment aligned | Every objective has an evaluation instrument | Rubric or formative assessment references the objective | Orphaned objective — taught but never measured |
| 3. Activities aligned | Every assessment criterion has activities that develop the skill | Activity verbs match or exceed assessment demand | Orphaned activity — practiced but never assessed |
| 4. Chain complete | Objective -> Assessment -> Activity links verified | No broken links | Broken chain — gaps in alignment |

### Common Backward Design Failures

| Failure | Description | Impact | Severity |
|---------|-------------|--------|----------|
| Assessment gap | Objective exists but no evaluation measures it | Student learning unmeasured | CRITICAL |
| Activity gap | Assessment criterion exists but no activity develops it | Students assessed on unpracticed skills | CRITICAL |
| Orphaned activity | Activity exists but connects to no assessed objective | Instructional time wasted | WARNING |
| Level mismatch | Activity develops at lower level than assessment demands | Students underprepared for evaluation | CRITICAL |

## Taxonomy Verification Rules

### Bloom Revisada — Verb-to-Level Mapping (Spanish)

| Level | Verbs (Indicative) | Common Misclassifications |
|-------|-------------------|--------------------------|
| Recordar | Listar, nombrar, identificar, reconocer, repetir, definir | "Identificar" can be Comprender if requires interpretation |
| Comprender | Explicar, describir, resumir, interpretar, clasificar, comparar | "Describir" can be Analizar if requires decomposition |
| Aplicar | Resolver, demostrar, usar, ejecutar, implementar, calcular | "Resolver" can be Analizar if problem is novel |
| Analizar | Diferenciar, organizar, atribuir, comparar, contrastar, examinar | "Comparar" can be Comprender if criteria are given |
| Evaluar | Juzgar, criticar, argumentar, justificar, valorar, defender | "Argumentar" is always Evaluar |
| Crear | Disenar, producir, elaborar, inventar, proponer, construir | "Elaborar" can be Aplicar if following a template |

### Marzano — Level Mapping

| Level | Description | Key Indicators |
|-------|-------------|----------------|
| Recuperacion | Recall and recognition | Student retrieves stored information |
| Comprension | Understanding meaning | Student explains in own words |
| Analisis | Extending knowledge | Student compares, classifies, creates analogies |
| Utilizacion | Applying in context | Student solves real-world problems |
| Metacognicion | Self-regulation | Student monitors own learning |
| Sistema del yo | Motivation and efficacy | Student sets goals, evaluates importance |

### Ambiguity Resolution Protocol

When a verb maps to multiple taxonomy levels, apply these rules:

1. **Context determines level**: "Describe the parts of a cell" (Recordar) vs. "Describe why photosynthesis matters for ecosystems" (Comprender/Analizar).
2. **Novelty determines level**: Applying a known procedure = Aplicar. Applying to a novel problem = Analizar.
3. **When still ambiguous**: Flag as WARNING with both possible levels and request human judgment.

## DUA Compliance Matrix

### Three Principles with Minimum Adaptations

| Principle | Description | Adaptation Examples | Minimum per Activity |
|-----------|-------------|-------------------|---------------------|
| Representation | Multiple means of accessing content | Visual diagrams, audio narration, text alternatives, manipulatives | At least 1 when principle applies |
| Engagement | Multiple means of motivation | Choice of tasks, collaborative option, self-assessment, real-world connection | At least 1 when principle applies |
| Action/Expression | Multiple means of demonstrating learning | Written, oral, visual, digital portfolio, physical demonstration | At least 1 when principle applies |

### Threshold Rule

Each activity must provide **at least 2 adaptations** across at least **2 of the 3 DUA principles**. Activities with only 1 adaptation or coverage of only 1 principle are non-compliant.

### Invalid DUA Adaptations (Do Not Count)

| Pseudo-Adaptation | Why Invalid |
|------------------|-------------|
| "Use visual resources" (unspecified) | Too vague — which visual resource? For what purpose? |
| "The teacher can adapt" | Delegates responsibility without providing adaptation |
| "Students can work in groups" (without structure) | Social arrangement is not a DUA adaptation unless it provides alternative access |
| "Use technology" (unspecified) | Technology is a medium, not an adaptation |

## Progressive Complexity Rules

Activities within a unit should follow this general pattern:

1. **Opening block**: Recordar/Comprender level — activate prior knowledge.
2. **Development blocks**: Comprender -> Aplicar -> Analizar — build and extend understanding.
3. **Closing block**: Analizar/Evaluar/Crear — synthesize and apply at higher levels.
4. **Evaluation**: Matches the highest level practiced in development.

### Acceptable Exceptions

- Brief recall activities within development blocks for scaffolding.
- Metacognitive prompts at any point (Marzano Metacognicion level).
- Emotional/motivational sections ("Mi emocion cuenta") not subject to taxonomy progression.
