Cómo Usar esta Biblioteca

Guía rápida para aprovechar 100% la potencia de NL-HP

Anatomía de un Prompt NL-HP

Cada prompt sigue una estructura de 4 capas:

  • ROL: Quién ejecuta (e.g., "discovery-conductor")
  • OBJETIVO: Qué se logra (e.g., "mapeo stakeholders completo")
  • PROTOCOLO: Cómo proceder (e.g., "iteración de entrevistas")
  • Parámetros: Controles ajustables ({MODO}, {FORMATO}, {PROFUNDIDAD})

Cadena de Dependencias

Los prompts no son independientes — respetan una secuencia de delivery:

  • Prerequisitos: Cada card indica qué entregables debe leer primero
  • Gates: 4 quality gates (G1, G1.5, G2, G3) bloquean progreso
  • Outputs: Los archivos/artefactos generados se heredan al siguiente prompt
  • Validación cruzada: Antes de cada gate se verifican inconsistencias

Parámetros & Personalización

Adapta cada prompt a tu contexto sin alterar el core protocol:

  • {MODO}: piloto-auto, desatendido, supervisado, paso-a-paso
  • {FORMATO}: markdown, html, docx, dual
  • {VARIANTE}: ejecutiva (~40%) o técnica (full)
  • {TIPO_SERVICIO}: enruta automáticamente agentes y skills
Protocolo Zero-Hallucination: Cada afirmación en un entregable está etiquetada con su fuente: [CÓDIGO], [CONFIG], [DOC], [INFERENCIA], [SUPUESTO]. Si más del 30% de un entregable es [SUPUESTO], se emite un banner de advertencia automático.
