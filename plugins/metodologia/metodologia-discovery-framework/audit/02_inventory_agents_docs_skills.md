## 5. Mapa de Relaciones

```
CLAUDE.md (hub)
  ├── references/ontology/ (13 archivos de conocimiento)
  │   ├── pipeline-orchestration.md → define fases → invoca skills
  │   ├── agent-committee.md → define agentes → asignados a fases
  │   ├── skills-catalog.md → lista skills → mapeadas a dominios
  │   ├── quality-gates.md → define gates → bloquean pipeline
  │   └── service-routing.md → routing → determina skills/agentes
  │
  ├── hooks/hooks.json → ejecuta scripts/
  │   ├── SessionStart → secrets-scan, scan-deliverables, session-context-gen,
  │   │                   auto-prime-check, ghost-menu-inject, session-changelog
  │   └── PostToolUse (Write|Edit) → session-changelog, context-prune, post-prime-calibrate
  │
  ├── scripts/lazy-load-resolver.sh → lee agents/ + skills/ por fase y tipo
  │
  ├── skills/ (111) → cada skill puede invocar otras skills
  │   ├── discovery-orchestrator → coordina TODAS las demás skills
  │   ├── output-engineering → coordina html-brand, brand-docx, brand-pptx, brand-xlsx
  │   └── design-system-brand → provee tokens a todas las skills de formato
  │
  └── commands/ (109) → invocan skills y scripts
```
