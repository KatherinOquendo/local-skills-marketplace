## Cloud Adoption Framework

Framework de gobernanza para adopcion cloud estructurada: [DOC]

1. **Strategy** — Definir motivacion, outcomes esperados
2. **Plan** — Inventario, racionalizacion, plan de migracion
3. **Ready** — Landing zones, networking, identity
4. **Adopt** — Migrate + Innovate
5. **Govern** — Policies, compliance, cost management
6. **Manage** — Operations, monitoring, optimization

## 7Rs de migracion

| R | Estrategia | Esfuerzo | Uso tipico |
|---|-----------|----------|------------|
| **Rehost** | Lift & shift | Bajo | Legacy con poco cambio |
| **Replatform** | Lift & optimize | Medio | Optimizar sin redisenar |
| **Refactor** | Re-architect | Alto | Modernizacion completa |
| **Repurchase** | Replace con SaaS | Variable | CRM, ERP → SaaS |
| **Retire** | Descomisionar | Bajo | Sistemas obsoletos |
| **Retain** | Mantener on-prem | Ninguno | Restricciones regulatorias |
| **Relocate** | Mover a otro cloud | Medio | Multi-cloud strategy |
