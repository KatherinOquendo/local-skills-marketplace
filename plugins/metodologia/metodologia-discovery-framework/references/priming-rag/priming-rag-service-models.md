# Universal Service Delivery Models — Priming RAG

## Insights

Service delivery in technology consulting follows well-established patterns that vary by engagement type. Understanding these models enables accurate scoping, realistic timeline estimation, and appropriate resource allocation during discovery. Each service type has distinct delivery characteristics that affect how the discovery pipeline should be configured.

## Key Points

- **10 service types** represent the universe of technology consulting engagements
- Each service type has a natural delivery cadence, sizing methodology, and success criteria
- Multi-Service engagements require program-level orchestration across workstreams
- Service type determines the AS-IS analysis variant and expert committee composition
- Hybrid delivery (onshore + nearshore + offshore) is common across all service types

## Content

### Software Development & Architecture (SDA)

**Delivery model:** Agile sprints (2-week), continuous delivery pipelines
**Sizing:** FTE-months + story points + technical complexity factors
**Key metrics:** Velocity, lead time, deployment frequency, change failure rate (DORA)
**Discovery focus:** Code quality, architecture patterns, technical debt quantification, modernization paths
**Typical engagement:** 3-18 months depending on scope

### Quality Assurance (QA)

**Delivery model:** Test factory with automation pipeline, continuous testing
**Sizing:** Test effort hours + automation coverage % + defect density targets
**Key metrics:** Test coverage, defect escape rate, automation ratio, mean time to detect
**Frameworks:** TMMi (Test Maturity Model integration), ISTQB certification levels
**Discovery focus:** Current test maturity, coverage gaps, automation opportunities, QA CoE design
**Typical engagement:** 6-24 months for QA transformation

### Project & Program Management

**Delivery model:** PMO-as-a-Service, methodology coaching, governance setup
**Sizing:** FTE-months + governance overhead + ceremony cost
**Key metrics:** On-time delivery %, budget variance, stakeholder satisfaction, velocity predictability
**Frameworks:** PMI/PMP, SAFe, Scrum@Scale, PRINCE2, Kanban
**Discovery focus:** Process maturity, methodology fitness, governance gaps, delivery predictability
**Typical engagement:** 3-12 months for PMO setup, ongoing for sustained management

### Robotic Process Automation (RPA)

**Delivery model:** Process assessment → bot development → hypercare → CoE
**Sizing:** Bot hours saved + ROI per process + automation complexity score
**Key metrics:** Processes automated, hours saved, error reduction, ROI per bot
**Frameworks:** IEEE 2755 (RPA best practices), process mining methodologies
**Discovery focus:** Process landscape mapping, automation potential scoring, platform selection
**Typical engagement:** 2-6 months per automation wave

### Data & Artificial Intelligence (Data-AI)

**Delivery model:** Data platform + model development + MLOps pipeline
**Sizing:** Model complexity + data volume + pipeline sophistication
**Key metrics:** Model accuracy, data quality score, time-to-insight, data coverage
**Frameworks:** DCAM (Data Management Capability Assessment Model), AI Adoption Lifecycle (Assess-Pilot-Scale-Optimize-Govern), CRISP-DM
**Discovery focus:** Data maturity assessment, AI readiness, use case portfolio prioritization
**Typical engagement:** 6-24 months for data platform, 2-6 months per AI use case

### Cloud Services

**Delivery model:** Assessment → migration waves → optimization → managed services
**Sizing:** Workload migration units + cloud spend optimization targets
**Key metrics:** DORA metrics, cloud spend efficiency, availability SLAs, migration velocity
**Frameworks:** AWS Well-Architected, Azure Cloud Adoption, Google Cloud Architecture
**Discovery focus:** Cloud readiness, workload classification (7R), FinOps baseline, security posture
**Typical engagement:** 3-18 months depending on portfolio size

### Staff Augmentation (SAS)

**Delivery model:** Profile matching → onboarding → integration → performance tracking
**Sizing:** Profiles x months x seniority level
**Key metrics:** Time-to-fill, retention rate, team velocity impact, cultural fit score
**Discovery focus:** Talent gap analysis, skills matrix, team scaling model, onboarding plan
**Typical engagement:** 3-24 months per team augmentation cycle

### UX Design

**Delivery model:** Research → design sprints → prototyping → testing → design system
**Sizing:** Design sprints + research cycles + usability testing rounds
**Key metrics:** System Usability Scale (SUS), task completion rate, time-on-task, error rate
**Frameworks:** Design Thinking (IDEO/Stanford), Jobs-to-be-Done, Lean UX
**Discovery focus:** Design maturity, user research capability, design system gaps, accessibility compliance
**Typical engagement:** 2-6 months for design system, ongoing for UX research

### Digital Transformation

**Delivery model:** Program-level orchestration across multiple workstreams
**Sizing:** Program-level estimation across all service types involved
**Key metrics:** Digital maturity index, change adoption rate, business outcome achievement
**Frameworks:** McKinsey Digital Quotient, Gartner Digital Business KPIs
**Discovery focus:** Transformation scope, organizational readiness, multi-workstream roadmap
**Typical engagement:** 12-36 months for enterprise-scale transformation

### Multi-Service

**Delivery model:** Integrated program with coordinated delivery across service types
**Sizing:** Composite of all involved service types with integration overhead (15-25%)
**Key metrics:** Cross-workstream dependency resolution, integrated timeline adherence
**Discovery focus:** Service integration points, shared resources, dependency management
**Typical engagement:** Duration of longest workstream + 20% coordination overhead
