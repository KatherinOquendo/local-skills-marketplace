## Caracteristicas del sector

| Dimension | Impacto en discovery |
|-----------|---------------------|
| **Regulacion** | Toda solucion debe considerar compliance (SFC, Basel, PCI-DSS, GDPR) |
| **Legacy systems** | Core bancarios de 20+ anos, COBOL, mainframes |
| **Seguridad** | Zero trust, encryption at rest/transit, SOC2 obligatorio |
| **Disponibilidad** | 99.99%+ required, disaster recovery mandatorio |
| **Datos sensibles** | PII, datos financieros — data governance estricta |
| **Time-to-market** | Presion por fintechs, necesidad de agilidad |

## Regulaciones clave

| Regulacion | Ambito | Impacto |
|-----------|--------|---------|
| **PCI-DSS** | Datos de tarjetas de pago | Requisito mandatorio para procesamiento de pagos |
| **Basel III/IV** | Capital y liquidez | Modelos de riesgo, reporte regulatorio |
| **GDPR/LGPD** | Proteccion de datos personales | Consentimiento, derecho al olvido, portabilidad |
| **Open Banking** | APIs obligatorias para datos financieros | APIs REST, OAuth2, consent management |
| **AML/KYC** | Anti-lavado, Conoce a tu Cliente | Identity verification, transaction monitoring |
| **SOX** | Controles financieros | Auditoria de TI, segregacion de funciones |
