Governance: `pipeline-governance` + `risk-controlling-dynamics`.

## OBJETIVO
Ejecutar discovery completo para servicios de UX/Design: evaluación de madurez UX, auditoría de experiencia, design system y accesibilidad. Si el usuario proporciona "$ARGUMENTS", usarlos como contexto del engagement.

## PROTOCOLO

### CP-0 · Ingesta
1. Recolecta: touchpoints digitales (web, mobile, tools internos), métricas UX, design system actual, brand guidelines.
2. Identifica nivel de madurez UX y dolor principal (conversión, adopción, accesibilidad, consistencia).
3. Clasifica: productos por criticidad, flujos por volumen de usuarios, componentes por reutilización.
4. Declara gaps. Si no hay métricas de usuario → recomienda analytics setup como prerequisito.

### CP-1 · Ejecución
1. **UX Maturity Assessment** — nivel 1 (ausente) a 5 (design-driven) según modelo NN/g.
2. **Heuristic Audit** — evaluación heurística de Nielsen sobre touchpoints principales.
3. **Accessibility Audit** — conformidad WCAG 2.1 nivel AA, automated + manual review.
4. **UX Metrics Baseline** — task completion rate, error rate, SUS score, NPS, CSAT, time-on-task.
5. **Design System Assessment** — existencia, adopción, coverage, actualización, tokenización.
6. **Research Practices Audit** — user testing, analytics usage, surveys, journey maps, personas.
7. **Pain Point Mapping** — dónde los usuarios abandonan, se frustran o fallan (cuantificado).
8. **Service Model Design** — UX audit, design system build, research ops, embedded team, coaching.
9. **UX Roadmap** — quick wins UI (M1), foundation design system (M2-M4), culture UX (M5-M12).

### CP-F · Validación
- [ ] UX maturity score con evidencia por dimensión
- [ ] Heuristic audit con severidad por hallazgo
- [ ] Accessibility audit con issues WCAG clasificados
- [ ] UX metrics baseline documentado
- [ ] Design system assessment con coverage %
- [ ] Pain points mapeados con impacto cuantificado
- [ ] Service model propuesto con justificación
- [ ] UX roadmap con quick wins y milestones
