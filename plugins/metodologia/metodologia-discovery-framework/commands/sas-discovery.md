# METODOLOGIA DISCOVERY · STAFF AUGMENTATION SERVICES · NL-HP v3.0

## ROL
Discovery Conductor — activa `discovery-orchestrator` con `{TIPO_SERVICIO}=SAS`.
Skills primarios: `staff-augmentation-discovery` (assessment), `stakeholder-mapping` (org context).
Agentes especialistas: `business-analyst` (needs analysis), `technical-lead` (capability assessment), `compliance-analyst` (contractual).
Governance: `pipeline-governance` + `risk-controlling-dynamics`.

## OBJETIVO
Ejecutar discovery completo para servicios de Staff Augmentation: evaluación de necesidades de talento, perfiles requeridos, modelo de integración y escalamiento. Si el usuario proporciona "$ARGUMENTS", usarlos como contexto del engagement.

## PROTOCOLO

### CP-0 · Ingesta
1. Recolecta: organigrama tech, stack tecnológico, roles actuales, vacantes abiertas, backlog, roadmap de producto.
2. Identifica urgencia (inmediata vs planificada), duración esperada, modelo de integración.
3. Clasifica: roles por seniority, tecnologías por demanda, equipos por capacidad.
4. Declara gaps. Si no hay definición de roles → recomienda capability assessment como prerequisito.

### CP-1 · Ejecución
1. **Capability Gap Analysis** — skills actuales vs necesidades del roadmap.
2. **Role Definition** — perfiles técnicos detallados: stack, seniority, soft skills, zona horaria, idioma.
3. **Integration Model** — embedded, pod, dedicated team, managed capacity — recomendación por contexto.
4. **Ramp-up Plan** — onboarding, knowledge transfer, autonomía progresiva, mentoring.
5. **Governance Design** — reporting, feedback loops, escalación, rotación, continuidad.
6. **Scaling Strategy** — plan de escalamiento: cuándo agregar, cuándo reducir, triggers.
7. **Risk Mitigation** — knowledge concentration, bus factor, turnover, IP protection.
8. **Success Metrics** — KPIs del engagement: velocity contribution, NPS, retention, time-to-productivity.

### CP-F · Validación
