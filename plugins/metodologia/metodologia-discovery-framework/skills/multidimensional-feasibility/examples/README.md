## Qué demuestran

1. **Claim Evidence Matrix** completa con 12 claims, tags de evidencia, niveles de confianza y semáforos de status.
2. **Análisis dimensional** de 3 de las 7 dimensiones (Investigación/TRL, Sistémica, Económica) — abreviados pero con calidad de referencia.
3. **Veredicto del Think Tank** con votación, disidencia documentada y condiciones suspensivas.
4. **Spike Catalog** con 3 spikes bien definidos: criterios de éxito/fracaso, timebox y alternativas.
5. **Dependency Graph** en Mermaid mostrando las relaciones entre claims, spikes y condiciones.

## Cómo usar como benchmark

Estos ejemplos son el **estándar mínimo de calidad** para el skill. Todo output del Think Tank debe igualar o superar:

- **Densidad de evidencia**: Toda claim tiene al menos 1 tag de evidencia. Claims con confianza < 3 disparan spikes.
- **Trazabilidad**: Cada claim conecta con una dimensión, un sabio y un veredicto dimensional.
- **Actionability**: Los spikes tienen criterios de éxito/fracaso binarios, timebox y alternativas explícitas.
- **Honestidad intelectual**: La disidencia se documenta, no se oculta. El veredicto reconoce limitaciones.

## Principio rector

La factibilidad NO es un sello de aprobación — es un análisis riguroso que puede (y debe) decir "no" cuando la evidencia lo exige. Un Think Tank que siempre dice "sí" no tiene valor.
