# FinOps — Example Outputs

Sample deliverables for the fictional **Acme Corp Digital Platform** FinOps assessment.

| File | Format | Description |
|------|--------|-------------|
| sample-output.md | Markdown | Primary 8-section FinOps assessment with Mermaid diagrams |
| sample-output.html | HTML | Branded version (MetodologIA Design System CSS) |
| sample-output.xlsx-spec.md | Spec | Excel workbook specification for cost data (6 sheets) |

## Format Guide

| Format | When to Use | Audience |
|--------|------------|----------|
| **Markdown** (default) | Internal analysis, discovery pipeline input, technical review | Engineering, Architecture |
| **HTML** | Client-facing presentations, executive reviews, stakeholder delivery | CTO, VP Engineering, Finance |
| **XLSX** | Finance-oriented cost analysis, budget planning, monthly reporting | Finance, FinOps Lead, CFO |
| **Dual** (md + html) | When both internal and client-facing versions are needed | Mixed audience |

## Key Conventions

- **Cost figures:** Always expressed as magnitude ranges (e.g., "$30K-$50K/mes"), NEVER exact prices
- **Evidence tags:** Every claim tagged with [CODIGO], [CONFIG], [DOC], [INFERENCIA], [SUPUESTO], [STAKEHOLDER]
- **Diagrams:** Mermaid embedded (quadrant chart for maturity, mindmap for taxonomy, gantt for roadmap)
- **Unit economics:** Always include cost-per-business-unit metrics, not just absolute spend
- **MetodologIA HTML branding:** Primary #122562, Dark #1F2833, Success #FFD700 (NEVER green for success states)
