# Data Mesh Strategy — Example Outputs

Sample deliverables for the fictional **Acme Corp Digital Platform** data mesh strategy.

| File | Format | Description |
|------|--------|-------------|
| sample-output.md | Markdown | Primary 7-section data mesh strategy with Mermaid diagrams |
| sample-output.html | HTML | Branded version (MetodologIA Design System CSS) |

## Format Guide

| Format | When to Use | Audience |
|--------|------------|----------|
| **Markdown** (default) | Internal analysis, discovery pipeline input, technical review | Data Architecture, Engineering Leads |
| **HTML** | Client-facing presentations, executive reviews, stakeholder delivery | CTO, VP Data, CDO |
| **Dual** (md + html) | When both internal and client-facing versions are needed | Mixed audience |

## Key Conventions

- **Readiness scoring:** 4 principles scored 1-5 with observable evidence and evidence tags
- **Evidence tags:** Every claim tagged with [CODIGO], [CONFIG], [DOC], [INFERENCIA], [SUPUESTO], [STAKEHOLDER]
- **Domain decomposition:** Based on DDD bounded contexts, not organizational chart
- **Data product specs:** Include SLAs (freshness, availability), quality dimensions, owner, consumers
- **Phased roadmap:** Always phased (pilot → expansion → maturity), never big-bang
- **Honest assessment:** Skill may recommend AGAINST data mesh when conditions are not met
- **MetodologIA HTML branding:** Primary #6366F1, Dark #1A1A2E, Success #22D3EE (NEVER green for success states)
- **Diagrams:** Mermaid embedded (quadrant for readiness, flowchart for domain map, gantt for roadmap)
