## Guia de Formato

### Markdown (`sample-output.md`)

- **Estructura:** TL;DR → 8 secciones (S1-S8) → Conclusiones
- **Tablas:** Rollout phases, communication calendar, training curriculum, resistance tactics, champion network, KPIs
- **Diagramas Mermaid:** Gantt (rollout timeline), flowchart (escalation path)
- **Adoption funnel:** Metricas por etapa con conversion rates
- **Kirkpatrick levels:** Training evaluado en 4 niveles

### HTML (`sample-output.html`)

- **Design System MetodologIA:** Primary #122562, Dark #1F2833, Success #FFD700
- **Nunca usar verde** para estados de exito — siempre amarillo (#FFD700)
- **Mermaid renderizado:** Via CDN v10 (`<pre class="mermaid">`)
- **Responsive:** Grid layout, tarjetas para champions e intervenciones
- **Impresion:** CSS `@media print` optimizado para A4

## Como Usar Estos Ejemplos

1. **Como referencia de calidad:** Verificar que outputs generados alcancen el nivel de detalle del sample
2. **Como template:** Copiar estructura de secciones y adaptar al proyecto especifico
3. **Como validacion:** Comparar output generado vs. sample para identificar gaps
4. **Continuidad con readiness:** Este output asume un readiness assessment previo (ver `change-readiness-assessment/examples/`)
